# 言台 0.3.0

言台 0.3.0 为事件、资源和帧路径增加可观测背压。高频提交保持有界，应用可以从统一
调试快照判断队列水位、帧替换与资源生命周期，而不依赖平台私有句柄。

## 新功能

- 平台协议提升到 1.2，能力查询新增`运行可观测性`，并报告`待呈现帧上限`为 1。
- 调试快照新增`事件队列`、`资源统计`和`帧统计`，覆盖累计计数、当前值与高水位。
- 窗口查询新增`待呈现帧`与`帧字节`，便于定位生产速度超过呈现速度的场景。
- 帧提交会主动唤醒事件循环；单槽替换避免连续动画或异步数据刷新积累无界帧队列。

## 修复内容

- 滚轮增量先完整验证两个轴再原子写回，任一轴溢出时不再部分污染已排队事件。
- 旧帧完成呈现时会核对内部代次，不会错误清除后续提交的新帧状态。
- 关闭含待呈现帧的窗口或应用会同步减少待处理计数，诊断值不会泄漏。

## 兼容性与平台

- 原生 ABI 仍为 v2，事件协议与绘制协议仍为 1.1；
- 平台协议从 1.1 提升为 1.2，只增加可忽略的能力、窗口和诊断字段；0.1/0.2 消费者
  无需迁移；
- 最低言序仍为 1.1.7，继续覆盖 Windows、macOS、Linux 的 x86_64 与 ARM64 六个目标。

## 安装示例

```toml
[依赖]
言台 = { 包 = "yanxu-platform", git = "https://github.com/yanxulang/yanxu-platform.git", 修订 = "v0.3.0", 版 = "^0.3" }

[权限]
图形界面 = true
原生扩展 = true
剪贴板 = true
文件对话框 = true
```

随后运行`yanbao 更`与`yanbao 查`，让言包选择当前目标的原生制品并验证摘要与大小。

## 升级方式

从 0.2.0 升级只需更新依赖修订和锁文件。既有方法返回值不变；读取能力、窗口查询或
调试快照的代码应继续忽略未知字段。诊断计数是进程内饱和累计值，不应持久化后再作为
下一次进程的基线。

## 制品与校验

Release 附带`yanxu-platform-0.3.0-six-targets.tar.gz`、同名`.sha256`和完整
`yanxu-platform-0.3.0.toml`。标签与归档包含同一批六目标动态库，清单逐项固定 SHA-256
和字节数；Release 只复用标签提交已经通过的 CI 候选。

## 第三方依赖

本版本不新增或升级运行时依赖，继续使用 0.2.0 已审计并精确锁定的窗口、表面、绘制、
文字、图片、剪贴板和文件对话框依赖。

## 已知限制

- 本版本提供呈现累计计数，但逐帧编号与呈现事件将在后续小版本定义；
- 事件离散溢出仍是致命错误，应用应及时处理批次并监控队列高水位；
- CPU 渲染、系统对话框、剪贴板和真实 IME 会话仍受目标桌面环境约束。
