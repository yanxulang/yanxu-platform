# 言台 1.0.0

言台 1.0.0 是首个稳定兼容版本。它不改变 0.9.0 的运行时方法、ABI 或协议组合，而是把
已经在六个桌面目标运行的边界提升为可自动验证的 1.x 契约，并明确支持、弃用、迁移和
发布政策。0.9.0 应用只需更新依赖与锁文件。

## 四层稳定契约

- 言序公开声明继续由`api/api-v1.json`固定，并由最低言序 1.1.7 重新生成比较。
- `api/native-abi-v2.json`固定 ABI v2 状态值、值标签、C 调用约定、结构布局、11 个函数
  签名、41 个操作和 5 个资源类型。测试从真实模块描述符读取名称、编号和调用指针。
- `api/protocol-contract-v1.json`固定平台 1.7、事件 1.3、无障碍 1.0 和绘制 1.1：包括
  50 个能力字段、39 个事件及合并方式、38 个角色、18 个状态、15 个动作、17 个绘制
  操作码、5 个路径操作码与全部主要硬上限。
- `api/error-codes-v1.json`登记实际运行时使用的 70 个`PLATFORM_*`错误码、类别、首次
  发布版本和稳定含义。测试扫描非测试运行时代码，防止遗漏、删除或无意改义。

1.x 可以在表尾或可忽略位置增加向后兼容内容，但必须提升对应次版本并更新契约。不兼容
C 布局、编号、字段类型、单位、所有权或既有成功调用语义只能进入新的 ABI、协议或包主
版本。应用应按错误码而不是可改善或本地化的消息文字分支。

## 生产支持边界

- 正式支持 Windows、macOS 和 Linux 的 x86-64/ARM64 六个目标；每个目标都构建 Release
  动态库、核对 ABI 导出，并运行言序集成、真实窗口和原生无障碍桥自动退出测试。
- 言序支持范围为`>=1.1.7, <2.0.0`。1.1.7 是每次发布的最低集成基线，尚未验收的 2.x
  宿主不会被清单静默接受。
- Rust 源码最低版本为 1.92。独立精确 1.92.0 作业执行格式、109 项测试和全目标严格
  Clippy，并与六目标、依赖策略和性能预算一起阻止不完整候选。
- 1.x 遵循语义化版本和明确弃用周期；高危漏洞需要收紧此前不安全输入时，修订版必须
  同时提供安全公告、稳定错误码和迁移方法。

## 安全、性能与长期运行

1.0 延续 0.9 已建立的生产门禁：绘制协议和 ABI 值畸形语料、4096 轮事件背压浸泡、
2048 轮严格配额资源生命周期浸泡，以及固定 Linux x86-64 Release 长样本性能预算。
性能报告验证绘制解码、事件合并与冲刷、资源生命周期和 CPU 栅格帧四项预算，并要求
中位绝对偏差低于 20%。

依赖策略继续以零公告忽略执行。2026-07-21 复核时，包含 PR #92 修复的 `fontdb` 正式
版本尚未发布，因此仍使用完整提交固定的已评审上游修复；当前依赖图不含停止维护的
`ttf-parser`。Wayland XML 解析链继续使用仓库内固定的最小安全补丁。正式版本出现后，
这些临时来源必须经相同门禁切回 crates.io。

## 从 0.x 升级

0.9.0 不需要源码迁移。更早版本应重点复核应用配额与退出状态机、事件背压、帧呈现反馈、
剪贴板容量、语义树与原生无障碍桥。完整逐版本清单和回滚方法见
[0.x 到 1.0 迁移指南](MIGRATION_0_X_TO_1_0.md)。

Git 依赖可以固定本标签：

```toml
[依赖]
言台 = { 包 = "yanxu-platform", git = "https://github.com/yanxulang/yanxu-platform.git", 修订 = "v1.0.0", 版 = "^1.0" }
```

应用仍需授予`图形界面`和`原生扩展`权限；剪贴板与文件对话框权限只在实际使用时授予。

## 发布制品

Release 附带`yanxu-platform-1.0.0-six-targets.tar.gz`、独立 SHA-256 文件、完整
`yanxu-platform-1.0.0.toml`和`yanxu-platform-1.0.0-performance.json`。归档与标签 Git
包包含同一批 Windows、macOS、Linux x86-64/ARM64 动态库，清单固定逐文件 SHA-256 与
大小，来源文档指向生成这些字节的成功六目标 CI。

本版本仍是桌面 CPU 二维后端，不承诺 GPU、移动端或 Web。稳定边界详见
[兼容政策](COMPATIBILITY.md)、[原生 ABI 冻结](NATIVE_ABI_V2.md)、
[协议冻结](PROTOCOL_CONTRACT_1_0.md)、[稳定错误码](ERROR_CODES_1_0.md)、
[安全架构](SECURITY.md)与[打包发布](PACKAGING.md)。
