# ADR-004：言据优先、JSON 兼容，帧数据使用原生值与二进制

- 状态：已接受
- 日期：2026-07-15

## 背景

GUI 的主题、样式和组件描述需要中文友好、可注释的人类可读配置；既有工具和言包工程
协议又广泛使用 JSON。事件与绘制是高频路径，不适合每次解析任一文本格式。

审计 `yanju 1.1.2` 后确认它已经提供稳定序列化、规范化输出、模式校验、流式读取、错误
位置以及言据/JSON 双向转换，因此本项目没有升级或修改言据。

## 决策

- 人工维护的主题、样式、组件描述、能力快照和协议示例优先使用言据。
- 对外工具入口保留 JSON；两种文本格式先转换到同一个言序典/列内存模型，再进入言台或
  言界逻辑。
- 未知字段在内存模型中保留，已知字段按显式模式校验；格式选择由文件扩展名或 CLI 明确
  参数决定，不用内容猜测覆盖用户意图。
- 不改变言包现有版本化 JSON 工程协议。
- 事件批次直接使用 ABI v2 原生值，绘制使用版本化二进制帧，资源使用句柄；言据和 JSON
  都不逐条进入渲染热路径。

## 后果

- 中文配置有一等格式，外部生态仍可交换 JSON。
- 同一模式和默认值只维护一次，格式转换可做往返测试。
- 字节串与原生资源不强塞进通用文本；持久化配置只引用资源路径或稳定标识。
- UI 仓库需要为两种输入增加等价夹具；平台仓库只消费已经解析的普通值。

## 放弃方案

- 只支持 JSON：不满足言据作为项目数据格式的定位。
- 只支持言据：会无理由破坏历史工具和通用生态。
- 每帧序列化言据/JSON：解析、分配和 ABI 调用量不适合交互渲染。
