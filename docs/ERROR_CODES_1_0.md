# 1.0 稳定错误码

言台 1.0 固定 70 个公开`PLATFORM_*`错误码。完整机器目录位于
[`api/error-codes-v1.json`](../api/error-codes-v1.json)，为每个代码记录类别、首次发布
版本和稳定含义。

应用必须按错误码而不是消息文字分支。错误码在 1.x 中不会删除、改名或复用为另一种
含义；错误消息只用于诊断，可以改善措辞或随宿主本地化。后续 1.x 次版本可以增加新代码，
因此消费者还必须保留未知代码的通用失败路径，不能把未知值当成成功。

目录测试扫描`native/src`中非测试运行时代码的全部字符串字面量，提取不以 `_`结尾的
`PLATFORM_*`代码，再与排序后的目录和 JSON 逐项比较。新增未登记代码、删除运行时代码、
重复条目或快照漂移都会使六目标测试失败。每个`since`值还依据已发布的 0.1.0–0.9.0
标签历史核对。

前缀可以用于遥测聚合，例如`PLATFORM_QUOTA_*`或`PLATFORM_DRAW_*`，但恢复逻辑应优先
匹配具体代码。是否重试还取决于调用和应用状态：容量、权限、无效输入、已关闭资源与
系统后端失败不能仅凭同一前缀采用相同策略。
