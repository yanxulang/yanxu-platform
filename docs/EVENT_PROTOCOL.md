# 事件协议 v1.3

言台一次跨 ABI 投递一个事件批次。协议值使用言序的典、列、文、数、理和空，不包含
平台句柄或原始指针。

## 批次封装

```yanxu
{
  「协议主」：1，
  「协议次」：3，
  「批次」：12，
  「事件」：【
    {「序号」：41，「类型」：「指针移动」，「时间」：1.284，
     「窗口」：2，「横坐标」：120.5，「纵坐标」：64.0，
     「来源」：「鼠标」，「设备」：「DeviceId(...)」，「修饰键」：{...}}
  】
}
```

`批次`在每个应用内递增；`序号`按事件产生顺序递增。高频合并可能使序号出现间隙。
`时间`是进程内单调秒数，只适合排序与计算间隔，不是墙上时钟。`窗口`是模型编号；应用
级事件为`空`。

## 事件名称

| 组 | v1.3 名称 |
| --- | --- |
| 应用 | `应用启动`、`应用激活`、`应用失活`、`退出请求` |
| 窗口 | `窗口显示`、`窗口隐藏`、`关闭请求`、`关闭完成`、`窗口移动`、`窗口缩放`、`DPI变化`、`获得焦点`、`失去焦点`、`需要重绘` |
| 呈现 | `帧呈现` |
| 无障碍 | `无障碍焦点请求`、`无障碍动作请求` |
| 指针 | `指针进入`、`指针离开`、`指针移动`、`指针按下`、`指针释放`、`指针取消`、`滚轮`、`手势` |
| 键盘 | `按键按下`、`按键释放`、`文本输入` |
| 输入法 | `IME组合开始`、`IME组合更新`、`IME组合提交`、`IME组合取消` |
| 文件 | `文件拖入`、`文件悬停`、`文件离开`、`文件放下` |
| 环境 | `系统主题变化`、`显示器变化`、`计时器` |

## 字段

所有事件都有公共字段，下面只列附加字段。系统不能提供某个可选值时，字段可以缺席；
消费者不得把未知字段当成错误。

| 事件 | 附加字段 |
| --- | --- |
| `窗口移动` | `横坐标`、`纵坐标`（逻辑像素） |
| `窗口缩放` | `宽`、`高`（逻辑像素），原生事件另带`物理宽`、`物理高` |
| `DPI变化` | `比例因子` |
| `指针进入`、`指针离开` | `来源`、`设备` |
| `指针移动` | `来源`、`设备`、`横坐标`、`纵坐标`、`修饰键` |
| `指针按下`、`指针释放` | 上述位置字段加`按钮` |
| 触摸/触控笔指针 | `指针编号`、`阶段`，可选`压力`、`高度角` |
| `滚轮` | `横滚`、`纵滚`、`单位`、`阶段`、`设备`、`修饰键` |
| `手势` | `手势`和按类型出现的`变化`、`横向变化`、`纵向变化`、`角度`、`压力`、`级别`或`轴`/`值` |
| `按键按下`、`按键释放` | `物理键`、`逻辑键`、`位置`、`修饰键`、`重复`、`合成`，可选`文本` |
| `文本输入` | `文本` |
| `IME组合更新` | `文本`，可选`选区起`、`选区终` |
| `IME组合提交` | `文本` |
| `文件悬停`、`文件放下` | `路径` |
| `系统主题变化` | `主题`，值为`浅色`、`深色`或`系统` |
| `计时器` | `计时器`资源编号 |
| `帧呈现` | `帧序号`、`提交时间`、`呈现时间`、`延迟毫秒` |
| `无障碍焦点请求` | `节点`、`树修订`、`来源` |
| `无障碍动作请求` | `节点`、`树修订`、`动作`、`参数`、`来源` |

`帧呈现`只在完整帧成功交给原生表面后产生。`呈现时间`与公共`时间`相同；它和
`提交时间`都使用`单调时间（）`的进程内秒数基准。`延迟毫秒`是两者之差，不能解释为
显示器扫描完成或墙上时间。被后续提交替换的待呈现帧不会产生该事件。

`无障碍焦点请求`只引用窗口当前语义树修订中存在、可见、启用、可聚焦且声明了`聚焦`
操作的节点。`来源`当前为`辅助技术`。事件本身不直接改写言界焦点；上层确认请求后更新
控件焦点并提交新的语义树。节点在处理前已消失时，上层忽略该请求，不得把编号复用于
同一棵树中的其他节点。

`无障碍动作请求`只引用同一修订中可见、启用且明确声明该动作的节点。`聚焦`不通过通用
动作事件发送，而使用上面的独立焦点请求。参数按动作固定：

| 动作 | `参数` |
| --- | --- |
| `设置值` | 输入类为不超过 65536 字节的文；滑块、滚动条和分割面板为有限数 |
| 输入类`选择` | `{「起」：非负整数，「终」：不小于起的整数}`，使用 UTF-8 字节边界 |
| `滚动` | 含`横向`或`纵向`有限增量的典 |
| 其他已声明动作 | `空` |

通用动作事件同样携带树修订；上层发现当前修订或节点身份已经变化时必须忽略，而不能把
旧参数应用到复用后的控件。畸形参数、未声明动作和不可交互节点在入队前被拒绝。密码框
的`设置值`参数属于敏感输入，上层不得写入日志或回填到语义树的`值`字段。

`修饰键`为：

```yanxu
{「Shift」：假，「Control」：假，「Alt」：假，「Super」：假，「主修饰」：假}
```

`主修饰`在 macOS 对应 Super/Command，在 Windows 与 Linux 对应 Control。物理键使用
稳定的 winit 键码名称（例如 `KeyA`），逻辑键表达布局后的含义（例如`a`或`Enter`）。
应用处理快捷键时用物理/逻辑键与修饰键；插入文字时只处理`文本输入`或 IME 提交，
不要从按键名称推导文字。

## 批处理与合并

队列默认上限为 4096。只有相邻、类型相同且窗口相同的事件参与合并：

- `指针移动`、`窗口移动`、`窗口缩放`、`需要重绘`保留最新事件；
- `滚轮`累加`横滚`与`纵滚`，其余字段取最新值；
- 所有离散事件都是合并屏障，按产生顺序保留；
- 队列装不下新的离散事件时返回`PLATFORM_QUEUE_FULL`并退出循环，不静默丢失。

事件循环在恢复、等待前、重绘和计时器处理后冲刷批次。回调执行完后调用 ABI v2 主机
泵，因此在回调中提交的新窗口状态会在下一次模型同步时生效。

## 关闭与退出

系统窗口关闭按钮先产生`关闭请求`，窗口仍然存在。若它是最后一个窗口，同一批次紧接着
产生应用级`退出请求`。应用可以：

- 调用`窗口.关闭（）`确认单窗关闭；
- 调用`应用.退出（）`结束整个事件循环；
- 什么也不做以取消请求。

模型资源实际销毁后产生`关闭完成`。这一分离保证上层可以实现未保存内容确认，而无需
后端内置对话框或控件逻辑。

## IME 顺序

典型中文输入为：`IME组合开始` → 一个或多个`IME组合更新` → `IME组合提交`；失焦、
取消或系统禁用时以`IME组合取消`结束。组合更新的选区是 UTF-8 字节范围。候选窗口位置
来自窗口最后一次配置的输入光标区域。详见 [TEXT_AND_IME.md](TEXT_AND_IME.md)。

## 兼容规则

- 主版本不同表示不兼容，启动时应拒绝组合。
- 次版本只增加事件、可选字段或合并类别，不改变既有字段含义。
- 消费者必须忽略未知事件名称和未知字段，并保持其余事件顺序。
- 生产者不得删除 v1 已定义的公共字段，也不得复用已有名称表达另一单位。
- 逻辑像素是稳定单位；需要诊断物理像素时读取明确带`物理`前缀的字段。
