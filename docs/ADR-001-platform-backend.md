# ADR-001：共享原生平台后端

- 状态：已接受
- 日期：2026-07-15

## 背景

言台首版必须覆盖 Windows x64/ARM64、macOS x64/ARM64 与 Linux x64/ARM64，同时向言序
暴露完全相同的窗口、事件、IME 和显示器语义。逐平台从零维护 Win32、AppKit、Wayland
与 X11 循环会显著扩大 unsafe 面和行为差异。

## 决策

使用固定版本 `winit 0.30.13` 作为原生窗口与事件循环抽象，使用 `softbuffer 0.4.8`
创建与窗口关联的 CPU 表面。Linux 同时编译 Wayland、X11 及动态加载特性。所有系统事件
在一个 `windowing.rs` 适配层归一化到版本化事件协议，系统对象不穿过 ABI v2。

窗口的期望状态保存在无句柄模型中；事件循环负责创建、同步和销毁真实窗口。这使绝大
多数生命周期、资源和协议测试不需要显示服务器。

## 后果

- 六目标共享代码与 CI 门禁，上层不需要 OS 条件分支。
- winit 能力是首版平台能力上限；缺失语义需要先贡献上游或增加隔离的小型平台适配器。
- Linux 必须持续验证 Wayland 可编译和 X11 真实窗口路径。
- 事件名称、单位与关闭语义由言台定义，不直接泄漏 winit 枚举。

## 放弃方案

- 分别直接实现 Win32/AppKit/Wayland/X11：控制力更高，但首版维护和 unsafe 成本过大。
- SDL/GLFW：窗口覆盖良好，但 IME、透明窗口、显示器和原生桌面语义不如当前组合贴合。
- Electron/WebView：不符合真正原生窗口且上层控件由言序实现的边界。
