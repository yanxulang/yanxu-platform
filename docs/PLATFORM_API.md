# 言台平台 API

本文件说明 `0.9.0` 的手写平台契约；由言序声明自动生成的签名索引见 [API.md](API.md)。
最低工具链是言序 `1.1.7`，原生接口使用 ABI v2。

## 协议与能力

`协议查询（）`返回：

```yanxu
{「平台主」：1，「平台次」：7，「事件主」：1，「事件次」：3，
 「无障碍主」：1，「无障碍次」：0，
 「绘制主」：1，「绘制次」：1，「ABI」：2}
```

`能力查询（）`返回原生窗口、多窗口、高 DPI、触摸、触控笔、IME、剪贴板、文件对话框、
文件拖放、CPU 二维绘制、复杂文字、`状态故障恢复`和`运行可观测性`；
`待呈现帧上限`为 1。帧能力明确报告`帧提交反馈`、`帧呈现反馈`、值为`进程内单调秒`的
`帧时间基准`和值为`帧呈现`的`动画驱动事件`。剪贴板能力还明确报告文字/图片支持、
文字字节上限、图片边长上限、图片字节上限和图片格式列。结果同时包含`系统`与`架构`，
Linux 还报告 Wayland 与 X11。无障碍能力报告语义树、焦点/动作请求、全部硬上限以及角色、
状态、动作集合；六个正式目标的`原生无障碍桥`为真，`原生无障碍后端`分别为`UIA`、
`NSAccessibility`或`AT-SPI`。1.7 还报告应用资源配额、硬上限、配额拒绝统计、生命周期
状态、退出幂等和关闭资源归零能力。调用者必须读取字段而不是根据操作系统名称猜测。

## 应用

```yanxu
定 应用 为 平台.应用（「应用名」，处理事件）；
应用.运行（）；       # 在当前 GUI 主线程阻塞运行事件循环
应用.唤醒（）；       # 唤醒正在等待的循环
应用.退出（）；       # 请求循环有序退出
应用.关闭（）；       # 关闭应用及其所有子资源
```

应用创建需要`图形界面`权限，包还必须允许 ABI v2 原生扩展。事件回调的唯一参数是事件
批次。`冲刷事件（）`可在循环运行前主动投递队列，返回是否投递了批次。一个应用实例
不能同时运行两个事件循环，也不能在事件循环已经退出后重新运行。生命周期按`就绪`、
`运行中`、`退出请求`、`已退出`、`已关闭`单向迁移；重复`退出（）`成功返回但只记录一次
有效请求，也不会重复唤醒事件循环。

应用级配额可在首个子资源或首次`运行（）`前下调：

```yanxu
定 上限：典 为 应用.配置资源配额（{
    「资源总数」：512，「窗口数」：16，「计时器数」：256，
    「图片数」：64，「字体数」：16，「图片字节」：67108864，
    「字体字节」：33554432，「帧字节」：33554432，
    「无障碍节点」：16384，「无障碍文字字节」：4194304
}）；
定 当前：典 为 应用.资源配额（）；
```

省略字段沿用当前值。未知字段、负数、`资源总数`为 0 或超过能力查询硬上限都会返回
`PLATFORM_QUOTA_CONFIG`；首次创建子资源或运行后配置返回`PLATFORM_QUOTA_LOCKED`。
资源数量、持有字节、全部窗口帧字节和全部窗口无障碍树总量都按应用合计，关闭资源会
归还当前使用量，但不会解冻配置。

应用还提供：

- `计时器（毫秒，重复）`；间隔必须为正，查询返回编号、间隔、重复和取消状态。
- `字体族（）`、`匹配字体（配置）`、`加载字体（字节串）`。
- `文字整形`、`文字测量`、`文字命中`。
- `加载图片（字节串）`；支持由 `image` 解码的 PNG/JPEG，最大边长 16384，解码预算不
  超过应用剩余图片字节。
- `显示器（）`与`系统主题（）`。
- `调试快照（）`；保留资源数量、运行状态与`状态恢复次数`，并返回：
  - `事件队列`：容量、当前、高水位、接收/合并/拒绝/批次/排出总数；
  - `应用状态`、`退出错误`和`应用生命周期统计`：运行、退出请求、重复退出、正常/失败
    退出、关闭、回收资源和资源归零总数；
  - `资源配额`：冻结状态、上限、使用量，以及总数/上限/配置/冻结和按配额类别的拒绝统计；
  - `资源统计`：当前、高水位、创建总数与关闭总数；
  - `帧统计`：待呈现及高水位、字节高水位、提交/替换/渲染/呈现/失败总数；
  - `无障碍统计`：当前树、节点与文字字节及其高水位，更新/去重/清除、焦点/动作请求和
    拒绝总数，以及原生桥当前激活、高水位、激活/停用、树同步、请求和原生拒绝总数。

诊断计数使用饱和整数，不是持久化格式，也不替代应用自身日志。

## 窗口

创建配置的已实现字段如下。未给出的字段使用表中默认值；未知字段被忽略，以允许配置
向前扩展。

| 字段 | 类型 | 默认值 |
| --- | --- | --- |
| `标题` | 文 | `言台窗口` |
| `宽`、`高` | 正数 | `800`、`600` |
| `可见` | 理 | 真 |
| `透明`、`无边框`、`置顶` | 理 | 假 |
| `最小尺寸`、`最大尺寸` | `[宽，高]` | 未设置 |
| `位置` | `[横坐标，纵坐标]` | 由系统决定 |

窗口方法`标题`、`大小`、`最小尺寸`、`最大尺寸`、`位置`、`显示`、`隐藏`、`最大化`、
`最小化`、`全屏`、`无边框`、`透明`、`置顶`和`请求重绘`都更新统一模型，再由事件循环
同步到系统窗口。`查询（）`返回编号、标题、逻辑客户区宽高、可见/窗口状态、比例因子、
所在显示器、是否已有帧、是否有待呈现帧、最后帧序号、待呈现帧序号、帧提交时间及当前
帧字节数。尚未提交帧时三个帧身份字段为`空`。

关闭请求不会自动销毁窗口。应用应在事件回调中决定调用`窗口.关闭（）`还是取消；最后
一个窗口的系统关闭同时产生应用级`退出请求`。调用`应用.退出（）`只退出循环，资源仍
由显式关闭或言序资源析构负责。

## 无障碍语义树

`窗口.更新无障碍（树）`原子校验并保存窗口语义树，返回修订、是否变化、节点数、文字
字节数和焦点节点；相同快照去重。`窗口.无障碍查询（）`返回同一摘要及规范化完整树，
`窗口.清除无障碍（）`幂等移除已有树。节点必须使用稳定正整数编号，且角色、状态、
操作、逻辑像素边界和子树全部受协议 1.0 校验。

0.8 在窗口首次显示前创建平台适配器，并把同一语义树接入 UIA、NSAccessibility 或
AT-SPI。树、标题、物理尺寸和 DPI 变化会同步到活动适配器；系统焦点和动作回到 GUI
事件循环后重新通过当前树修订、节点状态、参数及事件队列校验。输入框原生字符位置按
Unicode 字素转换为 UTF-8 字节选区；密码框不生成值或文字运行。消费者应同时检查
`无障碍语义树`、`原生无障碍桥`和`原生无障碍后端`；完整格式、映射、硬上限、动作参数
和错误代码见 [ACCESSIBILITY_PROTOCOL.md](ACCESSIBILITY_PROTOCOL.md)。

## 完整帧提交

`窗口.提交帧（命令列）`先调用`绘制编码`，再一次提交二进制帧；返回命令数。已经持有
二进制帧时可调用`提交缓冲`。需要追踪呈现时使用`提交帧反馈`或`提交缓冲反馈`，返回：

```yanxu
{「帧序号」：12，「命令数」：3，「替换」：真，「被替换帧」：11，「提交时间」：1.284}
```

首次提交或前一帧已呈现时，`替换`为假且`被替换帧`为空。后端在接收时完整验证帧头、
主版本、长度、填充和上限，只保存最后一帧。每个窗口只有一个待呈现槽；旧帧尚未呈现
时的新提交会替换它并增加`替换总数`，不会扩张队列。提交会唤醒事件循环；呈现用帧序号
清除状态，较慢的旧帧不会误清除较新的待呈现帧。具体命令见
[DRAW_PROTOCOL.md](DRAW_PROTOCOL.md)。

```yanxu
窗口.提交帧（【
    {「类型」：「清空」，「颜色」：【248，249，252，255】}，
    {「类型」：「圆角矩形」，「矩形」：【24，24，160，44】，
     「半径」：8，「颜色」：【36，99，235，255】}，
    {「类型」：「文字」，「位置」：【40，34】，「最大宽」：130，
     「字号」：18，「行高」：24，「字重」：600，
     「颜色」：【255，255，255，255】，「文本」：「确定」}
】）；
```

## 动画驱动契约

连续动画先在`需要重绘`事件中提交首帧，再以成功返回的`帧呈现`事件驱动下一帧。事件的
`帧序号`用于匹配提交回执；`呈现时间`作为下一次动画计算的单调时间，不能换算为墙上
时间。若提交回执的`替换`为真，上层必须丢弃与`被替换帧`绑定的等待状态，因为该帧不会
再产生呈现反馈。

`帧呈现`表示软件缓冲已成功交给原生表面，并不承诺显示器已经完成扫描或提供垂直同步
时间戳。窗口最小化、遮挡或表面尺寸为零时可能暂停反馈；恢复后以新的`需要重绘`事件
继续。支持平台协议 1.4 的消费者应优先使用该反馈链；兼容旧平台时才使用有界计时器，
且任一时刻仍只保留一个待呈现帧。

## 输入法与光标

`窗口.配置输入法（启用，光标区域，用途）`接受逻辑像素矩形`[x,y,宽,高]`；宽高允许
为零但不能为负。用途已实现`普通`、`密码`与`终端`。该矩形是系统候选窗定位依据。
`停止输入法（）`禁用 IME。`光标（名称，显示）`支持默认、链接/手型、文本、十字、
等待、进度、移动、禁止、抓取以及四类调整尺寸光标。

完整组合事件和文字索引规则见 [TEXT_AND_IME.md](TEXT_AND_IME.md)。

## 系统服务

剪贴板函数需要`剪贴板`权限。文字接口只处理 UTF-8，读写上限均为 16 MiB：

```yanxu
平台.剪贴板写入（「内容」）；
定 内容：文|空 为 平台.剪贴板读取（）；
```

图片接口只接受 RGBA8，每个像素依次为红、绿、蓝、透明度四个字节：

```yanxu
引「标准:字节」为 字节；

平台.剪贴板写入图片（{
    「格式」：「RGBA8」，「宽」：1，「高」：1，
    「内容」：字节.从数列（【35，99，235，255】）
}）；
定 图片：典|空 为 平台.剪贴板读取图片（）；
```

图片宽高必须为 1..16384，内容必须严格等于`宽 × 高 × 4`字节且不得超过 256 MiB。
读取到的系统图片同样经过验证。格式或形状错误返回`PLATFORM_CLIPBOARD_IMAGE`，任一
容量上限或尺寸乘法溢出返回`PLATFORM_CLIPBOARD_LIMIT`。调用前应读取能力字段，不把
这些上限硬编码为其他后端的默认值。

文件对话框需要`文件对话框`权限。打开、保存和选择目录都接受配置典：`标题`、`目录`、
`文件名`和`过滤器`；每个过滤器为`{「名称」：文，「扩展」：列<文>}`。用户取消时返回
空，多选函数成功时返回路径列。路径按宿主可表示的 Unicode 文本返回。

## 字体与文字

文字配置字段为`字族`、`字重`（1..1000）、`斜体`、`字号`、`行高`、`最大宽`和`换行`。
默认值分别为系统无衬线、400、假、16、22、无限宽和真。整形返回每行的顶部、基线、
行高和字形范围，以及每个字形的实际回退字族、字重、斜体、字形编号、位置、宽度、
方向和原文 UTF-8 字节范围。

`文字命中`返回 UTF-8 字节边界，可直接作为同一原文的切片位置。字形输出可以按字族、
字重和斜体分组后作为`字形序列`绘制，从而保证光标布局与绘制使用同一整形结果。

## 图片与显示器

`图片.查询（）`返回编号、宽和高。绘制命令引用图片原生资源，呈现时才解析句柄；关闭
后的图片句柄不会被继续读取。显示器记录名称（可空）、物理位置、物理尺寸、比例因子和
主显示器标志。窗口查询中的宽高始终是逻辑像素。

## 错误模型

原生错误通过 ABI v2 的稳定代码返回，消息仅用于诊断。完整 1.0 机器目录与兼容规则见
[1.0 稳定错误码](ERROR_CODES_1_0.md)。调用者可以按下列前缀归类遥测：

| 代码或前缀 | 含义 |
| --- | --- |
| `PLATFORM_PERMISSION_*` | 缺少 GUI、剪贴板或文件对话框权限 |
| `PLATFORM_ARGUMENT_COUNT`、`PLATFORM_VALUE_*` | 参数数量、类型或范围错误 |
| `PLATFORM_RESOURCE_CLOSED`、`PLATFORM_RESOURCE_TYPE` | 资源已关闭或类型不符 |
| `PLATFORM_WRONG_THREAD` | 资源不属于当前 ABI 事件循环/所有者线程 |
| `PLATFORM_EVENT_LOOP*`、`PLATFORM_WINDOW_CREATE` | 事件循环或系统窗口失败 |
| `PLATFORM_APPLICATION_EXITED` | 已退出应用拒绝再次运行 |
| `PLATFORM_QUOTA_*` | 应用资源配额耗尽、配置无效或已经冻结 |
| `PLATFORM_QUEUE_FULL` | 离散事件超过有界队列，循环停止而不是丢事件 |
| `PLATFORM_DRAW_*` | 绘制主版本、损坏、UTF-8、数值或容量错误 |
| `PLATFORM_FRAME_SEQUENCE` | 单窗口内部帧代次已耗尽，拒绝继续覆盖 |
| `PLATFORM_ACCESSIBILITY_*` | 无障碍树、上限、编号、角色、状态、动作、焦点或修订错误 |
| `PLATFORM_TEXT_*`、`PLATFORM_FONT_INVALID` | 文字选项、容量或字体文件错误 |
| `PLATFORM_IMAGE_INVALID` | 图片格式或硬尺寸无效；应用容量不足使用`PLATFORM_QUOTA_IMAGE_BYTES` |
| `PLATFORM_CLIPBOARD_IMAGE`、`PLATFORM_CLIPBOARD_LIMIT` | 剪贴板图片形状/格式或文字、图片容量错误 |
| `PLATFORM_CALLBACK_*` | 回调已释放、投递或泵执行失败 |
| `PLATFORM_BACKEND_PANIC` | 后端 panic 已在 FFI 边界隔离 |

资源和线程规则见 [RESOURCE_LIFETIME.md](RESOURCE_LIFETIME.md) 与
[THREADING_MODEL.md](THREADING_MODEL.md)。
