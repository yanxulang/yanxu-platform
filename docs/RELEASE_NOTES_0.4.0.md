# 言台 0.4.0

言台 0.4.0 把系统剪贴板升级为有界、可协商的数据接口。除原有 UTF-8 文字外，应用现在
可以用统一 RGBA8 结构读写图片，并在调用前查询格式和容量上限。

## 新功能

- 新增`剪贴板读取图片（）`与`剪贴板写入图片（图片）`，数据形状为
  `{「格式」：「RGBA8」，「宽」：数，「高」：数，「内容」：字节串}`。
- 平台协议提升到 1.3；能力查询新增文字/图片支持、文字字节上限、图片边长上限、图片
  字节上限和图片格式列。
- 文字与图片读取结果也执行容量验证，系统剪贴板不能绕过言台的公开边界。

## 安全与可靠性

- UTF-8 文字读写上限为 16 MiB，超限返回`PLATFORM_CLIPBOARD_LIMIT`。
- RGBA8 图片单边上限为 16384、内容上限为 256 MiB，且字节数必须严格等于
  `宽 × 高 × 4`。
- 图片尺寸使用检查乘法；格式或形状错误返回`PLATFORM_CLIPBOARD_IMAGE`，容量或溢出
  返回`PLATFORM_CLIPBOARD_LIMIT`。

## 兼容性与平台

- 原生 ABI 仍为 v2，事件协议与绘制协议仍为 1.1；
- 平台协议从 1.2 提升为 1.3，只增加可忽略的能力字段和新方法；原有文字方法签名不变；
- 最低言序仍为 1.1.7，继续覆盖 Windows、macOS、Linux 的 x86_64 与 ARM64 六个目标。

## 安装示例

```toml
[依赖]
言台 = { 包 = "yanxu-platform", git = "https://github.com/yanxulang/yanxu-platform.git", 修订 = "v0.4.0", 版 = "^0.4" }

[权限]
图形界面 = true
原生扩展 = true
剪贴板 = true
文件对话框 = true
```

随后运行`yanbao 更`与`yanbao 查`，让言包选择当前目标的原生制品并验证摘要与大小。

## 升级方式

从 0.3.0 升级需更新依赖修订、次版本约束和锁文件。原有文字剪贴板调用无需改名，但超过
16 MiB 的输入现在会被明确拒绝。使用图片接口前应检查平台次版本或能力字段，并按能力
返回的格式与上限构造数据。

## 制品与校验

Release 附带`yanxu-platform-0.4.0-six-targets.tar.gz`、同名`.sha256`和完整
`yanxu-platform-0.4.0.toml`。标签与归档包含同一批六目标动态库，清单逐项固定 SHA-256
和字节数；Release 只复用标签提交已经通过的 CI 候选。

## 第三方依赖

本版本保持 `arboard 3.6.1` 不变并启用其`image-data`功能。新增传递图像编解码辅助依赖
均由锁文件固定，并通过与其余六目标依赖相同的公告、来源和许可门禁。

## 已知限制

- 图片剪贴板当前只承诺 RGBA8，不提供 HTML、富文本、文件列表或任意 MIME 类型；
- 系统剪贴板仍依赖真实桌面用户会话；无头环境只能验证确定性参数和错误路径；
- 读写是同步平台操作，应在 GUI 所有者线程调用并避免在高频事件处理中传输大图片。
