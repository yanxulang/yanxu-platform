# 言台 0.2.0

言台 0.2.0 强化 ABI 与共享状态故障隔离，使一次已捕获的后端恐慌不会通过中毒锁继续影响
后续资源清理、文字服务或事件循环。

## 新功能

- 模型、文字服务与事件循环代理统一使用可恢复互斥锁；恢复会清除中毒标记并累计诊断计数。
- 平台协议提升到 1.1，`能力查询（）`新增`状态故障恢复`，`应用.调试快照（）`新增
  `状态恢复次数`。

## 修复内容

- ABI 调度在执行前把输出值和错误值初始化为空，失败路径不会遗留调用方上一次调用的资源值。
- 资源析构、事件冲刷、计时器、文字、图片和窗口状态路径不再使用会因锁中毒再次恐慌的访问方式。

## 兼容性与平台

- 原生 ABI 仍为 v2，事件协议仍为 1.1，绘制协议仍为 1.1；
- 平台协议从 1.0 提升为 1.1，只增加可忽略的能力与诊断字段；0.1 消费者无需迁移；
- 最低言序仍为 1.1.7，继续覆盖 Windows、macOS、Linux 的 x86_64 与 ARM64 六个目标。

## 安装示例

```toml
[依赖]
言台 = { 包 = "yanxu-platform", git = "https://github.com/yanxulang/yanxu-platform.git", 修订 = "v0.2.0", 版 = "^0.2" }

[权限]
图形界面 = true
原生扩展 = true
剪贴板 = true
文件对话框 = true
```

随后运行`yanbao 更`与`yanbao 查`，让言包选择当前目标的原生制品并验证摘要与大小。

## 升级方式

从 0.1.0 升级只需更新依赖修订和锁文件。现有应用、窗口、事件、文字、绘制、图片和系统
服务 API 不变；读取协议或诊断典的代码应继续忽略未知字段。

## 制品与校验

Release 附带`yanxu-platform-0.2.0-six-targets.tar.gz`、同名`.sha256`和完整
`yanxu-platform-0.2.0.toml`。标签与归档包含同一批六目标动态库，清单逐项固定 SHA-256
和字节数；Release 只复用标签提交已经通过的 CI 候选。

## 第三方依赖

本版本不新增或升级运行时依赖，继续使用 0.1.0 已审计并精确锁定的窗口、表面、绘制、
文字、图片、剪贴板和文件对话框依赖。

## 已知限制

- 1.0 路线当前仍使用 CPU 二维绘制，不承诺 GPU、移动端或 Web 后端；
- 原生无障碍桥将在后续小版本提供，0.2.0 尚未把上层语义树接入系统；
- 文件对话框、剪贴板和真实 IME 会话仍依赖目标桌面环境。
