# 无障碍协议 v1.0

言台用纯言序值接收每个窗口的语义树，不把 UIA、NSAccessibility、AT-SPI 对象或平台
句柄带过 ABI。1.0.0 保持协议 1.0 的有界树、校验、查询和焦点/动作事件契约，并通过
统一适配层把同一模型注册到 Windows UIA、macOS NSAccessibility 与 Linux AT-SPI。
调用方从`原生无障碍桥`和`原生无障碍后端`能力字段协商实现，不按操作系统名称猜测。

## 节点格式

```yanxu
{
  「编号」：2，
  「角色」：「按钮」，
  「名称」：「保存」，
  「描述」：「保存当前内容」，
  「值」：空，
  「状态」：{「启用」：真，「可见」：真，「可聚焦」：真，「焦点」：真}，
  「操作」：【「点击」，「聚焦」】，
  「边界」：【12，12，80，32】，
  「子」：【】
}
```

`编号`、`角色`和`边界`必需。编号是窗口内正整数，同一快照中不得重复；更新之间只要
控件身份未变就必须保持稳定。`名称`、`描述`默认为空文，`值`默认为空，`状态`默认为
空典，`操作`与`子`默认为空列。未知节点字段会被忽略，以允许同一主版本增加可选信息。

`值`只允许空、理、整数、有限数或有界文。`边界`为逻辑像素`[横坐标,纵坐标,宽,高]`；
坐标绝对值不超过 1000000，宽高为 0..1000000 的有限数。隐藏节点可以省略出树；若保留，
必须用`可见：假`明确表达。`密码框`的`值`必须为空，不能把口令或掩码文字写进可查询
快照；辅助技术设置的新值只通过有界动作参数交给上层。

## 角色

协议 1.0 支持：

- 结构：`窗口`、`组`、`面板`、`控件`、`弹出层`、`对话框`、`工具栏`、`分隔符`；
- 内容：`文字`、`标题`、`链接`、`图片`、`画布`；
- 表单：`按钮`、`复选框`、`单选框`、`切换`、`输入框`、`多行输入框`、`密码框`、
  `组合框`、`滑块`、`进度条`、`滚动条`；
- 集合：`列表`、`列表项`、`树`、`树项`、`标签页`、`标签`、`菜单栏`、`菜单`、`菜单项`、
  `表格`、`行`、`单元格`、`滚动容器`、`分割面板`。

未知角色返回`PLATFORM_ACCESSIBILITY_ROLE`，不会降级为通用节点。调用者可从
`能力查询（）【「无障碍角色」】`读取当前完整集合。

## 状态

通用理状态为`启用`、`可见`、`可聚焦`、`焦点`和`忙碌`。其余状态按角色限制：

| 状态 | 适用角色或值 |
| --- | --- |
| `只读`、`必填`、`无效` | 文字输入、组合框及可编辑/可检查表单角色 |
| `选中`、`当前` | 列表项、树项、标签、菜单项、行、单元格、单选框 |
| `已检查` | 复选框、单选框、切换、菜单项 |
| `混合` | 复选框；不能与`已检查：真`同时出现 |
| `展开` | 组合框、树项、菜单、菜单项、弹出层 |
| `多选` | 列表、树、标签页、表格 |
| `模态` | 对话框、弹出层 |
| `按下` | 按钮、切换 |
| `方向` | 滑块、滚动条、分割面板、工具栏、分隔符；值为`横向`或`纵向` |
| `级别` | 标题、树项、行；整数 1..64 |

一棵树最多有一个`焦点：真`节点。该节点必须同时明确`可聚焦：真`，且不能不可见或
禁用。状态名、类型、角色组合或焦点不变量错误分别返回稳定状态/焦点错误。

## 操作与事件

节点只能声明与角色匹配的`聚焦`、`点击`、`设置值`、`选择`、`取消选择`、`复制`、
`剪切`、`粘贴`、`展开`、`折叠`、`增加`、`减少`、`滚动`、`滚动到`和`显示菜单`。
原生桥只能为当前修订中仍可见、启用并声明了对应操作的节点排队请求。

`聚焦`使用离散的`无障碍焦点请求`事件；其他操作使用`无障碍动作请求`。事件携带窗口、
节点编号、树修订和来源，动作事件另带动作与已校验参数。参数形状和失效处理见
[EVENT_PROTOCOL.md](EVENT_PROTOCOL.md)。事件不会直接改写上层控件状态；言界处理成功后
提交新快照，使系统桥看到最终状态。

## 原生桥映射

原生窗口使用保留编号 0 作为 AccessKit 根；协议节点编号全部为正整数，因此不会冲突。
适配器必须在窗口首次显示前创建。辅助技术首次激活时，从线程安全快照同步返回完整树；
之后语义树修订、窗口标题、客户区物理尺寸或 DPI 比例变化时，活动适配器接收新的完整树。
所有系统窗口事件先交给适配器，再进入言台事件归一化。

协议逻辑像素边界乘当前 DPI 比例后交给原生 API。38 个角色和适用状态映射为平台通用
语义；不可见、禁用、忙碌、只读、必填、无效、选中、展开、混合、模态、方向与级别均
保留。输入框和多行输入框生成稳定的原生文字运行；Unicode 字素长度保留 UTF-8 字节数，
系统选择范围可安全还原为协议的`起`/`终`字节边界。异常长的单个字素只在原生字符模型
中回退为 Unicode 标量，字节总量和边界仍保持一致。

原生操作通过事件循环代理回到模型。标准焦点、点击、设置值、选择、展开/折叠、增减、
滚动、滚动到与上下文菜单使用平台动作；取消选择、复制、剪切、粘贴及同时存在的点击/
选择使用带稳定编号的自定义动作。滚动项转换为单位增量，滚动页转换为节点逻辑宽或高。
任何请求都在同一模型锁内重新查询当前树并调用协议 1.0 校验，不能绕过修订、节点状态、
动作声明、参数或事件队列上限。

密码框继续不暴露`值`，也不生成原生文字运行。系统仍可请求有界`设置值`，但 1.0.0 不
向原生辅助技术声明密码文字选区；这避免为转换选区而复制、推断或记录口令内容。

## 更新、查询与修订

```yanxu
定 结果：典 为 窗口.更新无障碍（树）；
# {「修订」：1，「变化」：真，「节点数」：2，「文字字节」：30，「焦点」：2}

定 快照：典 为 窗口.无障碍查询（）；
# 上述摘要另含「树」

窗口.清除无障碍（）；
```

第一次有效更新把修订设为 1；不同快照单调递增，相同规范化快照返回`变化：假`且不增加
修订。清除已有树算一次变更，重复清除幂等。修订限定在言序整数范围，耗尽时返回
`PLATFORM_ACCESSIBILITY_REVISION`，不会回绕。更新只返回常量大小摘要，完整树只在显式
查询时返回。

## 硬上限

| 项目 | 上限 |
| --- | ---: |
| 单窗口节点 | 16384 |
| 嵌套深度（根为 0） | 64 |
| 单节点直接子节点 | 4096 |
| 单节点操作 | 16 |
| 单个名称、描述或文字值 | 65536 UTF-8 字节 |
| 单窗口名称、描述和文字值总量 | 4194304 UTF-8 字节 |

这些值也由能力查询逐项返回。达到任一上限时整次更新失败并保留旧树，不接受截断树。

## 稳定错误

| 代码 | 含义 |
| --- | --- |
| `PLATFORM_ACCESSIBILITY_TREE` | 必需字段、值类型或边界无效 |
| `PLATFORM_ACCESSIBILITY_LIMIT` | 节点、深度、子、操作或文字超过硬上限 |
| `PLATFORM_ACCESSIBILITY_DUPLICATE` | 同一快照出现重复编号 |
| `PLATFORM_ACCESSIBILITY_NODE` | 请求引用当前修订中不存在的节点 |
| `PLATFORM_ACCESSIBILITY_ROLE` | 角色未知 |
| `PLATFORM_ACCESSIBILITY_STATE` | 状态名称、类型或角色组合无效 |
| `PLATFORM_ACCESSIBILITY_ACTION` | 操作、角色、可交互状态或参数无效 |
| `PLATFORM_ACCESSIBILITY_FOCUS` | 焦点不唯一或目标不可聚焦 |
| `PLATFORM_ACCESSIBILITY_REVISION` | 修订计数耗尽 |

离散请求仍受事件队列上限约束；队列已满时返回`PLATFORM_QUEUE_FULL`，不会静默丢失。

## 兼容规则

- 无障碍主版本不同表示树和动作契约不兼容；消费者必须拒绝组合。
- 同主版本的次版本只能增加角色、状态、动作或可选节点字段。
- 生产者提交前读取能力集合，不把较新角色伪装成较旧但语义不同的角色。
- 节点编号只在窗口和树修订语境中有效，不是原生句柄，也不能跨窗口比较。
- 平台协议 1.6 只增加原生桥能力、后端名称和诊断字段；无障碍协议仍为 1.0。
- 平台协议 1.7 增加应用级无障碍节点与文字总配额；单窗口树格式、硬上限和动作语义不变。
