# ADR-003：文字布局以 UTF-8 索引和可重放字形表示

- 状态：已接受
- 日期：2026-07-15

## 背景

代码编辑器需要可靠的光标、选区、双向文本和字体回退。只提供“测量字符串/绘制字符串”
无法说明连字、组合字符和一个原文簇对应多个字形时的索引关系。

## 决策

文字 API 接收 UTF-8 文本与字族、字重、斜体、字号、行高、宽度和换行配置。整形结果以
原文 UTF-8 字节半开范围表达簇，返回每行指标和每个字形的实际回退字族、字重、斜体、
字形编号、位置、宽度与方向。命中测试也返回 UTF-8 字节边界。

绘制协议同时保留方便标签使用的字符串命令，以及可直接重放上述结果的`字形序列`。回退
结果按实际字体属性分组，从而让布局和绘制选择同一字体面。

## 后果

- 言序切片与持久化原文都能使用同一索引单位，不需要 UTF-16 平台特例。
- 消费者必须尊重簇边界，不能把字节索引当字符计数。
- 字形编号只在对应字体面内稳定；跨机器重放需携带同一字体文件。
- 简单控件仍可用字符串绘制，不强迫所有调用者手工管理字形。

## 放弃方案

- Unicode 标量或“字符序号”：无法无损表达 UTF-8 切片和复杂簇。
- UTF-16 索引：偏向部分系统 API，会给言序字符串增加转换层。
- 只返回总宽高：不足以实现编辑器光标、选择和命中测试。
