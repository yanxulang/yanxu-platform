# 1.0 协议冻结

言台 1.0 的机器协议契约位于
[`api/protocol-contract-v1.json`](../api/protocol-contract-v1.json)。它固定当前四层协商
边界：

- 平台协议 1.7 的查询字段、能力字段名称与类型，以及应用资源硬上限字段和值；
- 事件协议 1.3 的批次封装、公共字段、39 个事件名称、默认队列容量和合并方式；
- 无障碍协议 1.0 的 38 个角色、18 个状态、15 个动作与全部树硬上限；
- 绘制协议 1.1 的魔数、字节序、头布局、对齐、17 个操作码、5 个路径操作码和容量上限。

六目标测试从实际`协议查询`与`能力查询`结果、事件批次编码、事件种类、无障碍词汇、
绘制操作常量和执行路径上限重新生成 JSON，并与快照逐字节比较。编码器与渲染器分别
使用的路径、字形上限也必须相等。这样可以发现版本值未更新、字段删除或改型、事件重命名、
合并语义漂移、操作码重排和生产者/消费者上限不一致。

同一主版本内只能做向后兼容扩展：新增平台能力、事件、无障碍词汇或绘制操作时，必须
提升相应次版本、只在表尾或可忽略位置追加、更新快照和手写协议文档，并证明旧消费者能
忽略新增内容。删除、复用名称或编号、改变既有字段类型/单位、收紧已承诺语义，必须提升
对应协议主版本并提供迁移说明。

机器快照用于漂移门禁，不替代字段含义、动作参数和使用顺序说明。完整语义仍分别见
[平台 API](PLATFORM_API.md)、[事件协议](EVENT_PROTOCOL.md)、
[无障碍协议](ACCESSIBILITY_PROTOCOL.md)与[绘制协议](DRAW_PROTOCOL.md)。
