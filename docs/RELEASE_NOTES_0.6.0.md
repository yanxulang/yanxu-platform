# 言台 0.6.0

言台 0.6.0 为言界到系统辅助技术之间建立生产级、跨平台且有界的语义协议。窗口可以
提交稳定编号的语义树，言台统一验证角色、状态、动作、焦点、边界和动作参数，并定义
后续原生桥使用的焦点与动作请求事件。

## 新功能

- 新增`窗口.更新无障碍（树）`、`窗口.无障碍查询（）`和`窗口.清除无障碍（）`。
- 语义树更新返回修订、变化状态、节点数、文字字节数和焦点；相同规范化快照去重，
  清除幂等。
- 无障碍协议 1.0 定义 38 个角色、18 个状态、15 个动作及角色组合规则。
- 事件协议 1.3 定义`无障碍焦点请求`和`无障碍动作请求`，事件绑定窗口、节点与树修订。
- 调试快照新增当前树/节点/文字、高水位、更新/去重/清除、焦点/动作请求和拒绝计数。

## 有界树与校验

每个窗口最多保存 16384 个节点、根为 0 时最大 64 的嵌套深度和 4194304 UTF-8 文字
字节；单节点最多有 4096 个直接子节点、16 个动作，单个名称、描述或文字值最多 65536
字节。边界只接受有限逻辑像素，节点编号必须为窗口内不重复的正整数。

角色、状态和动作不是自由文本。无效组合、重复编号、第二个焦点、隐藏/禁用焦点、
未声明动作、畸形设置值/选区/滚动参数和全部容量错误都会拒绝整次操作并保留旧树。
密码框的语义值必须为空，防止口令进入查询和诊断路径。

## 焦点与动作契约

原生桥只能为当前树修订中仍存在、可见、启用并声明相应能力的节点排队请求。聚焦使用
独立焦点事件，其他动作使用通用动作事件；离散事件受既有 4096 队列背压保护，不会静默
丢失。事件本身不直接更改上层控件，言界处理成功后提交新树来确认最终状态。

0.6.0 的`原生无障碍桥`能力明确为假。本版本冻结共享模型和事件边界，尚不把树注册到
Windows UIA、macOS NSAccessibility 或 Linux AT-SPI；系统桥将在后续版本单独启用和
验收。

## 协议与兼容性

- 原生 ABI 仍为 v2，绘制协议仍为 1.1。
- 平台协议从 1.4 提升到 1.5，事件协议从 1.2 提升到 1.3，新增无障碍协议 1.0。
- 既有窗口、帧、剪贴板、文字和事件字段的含义不变；旧消费者忽略新增事件即可继续
  工作。
- 新消费者必须同时检查`无障碍语义树`与`原生无障碍桥`，不能仅按操作系统猜测能力。
- 最低言序仍为 1.1.7，继续覆盖 Windows、macOS、Linux 的 x86_64 与 ARM64 六个目标。

## 安装示例

```toml
[依赖]
言台 = { 包 = "yanxu-platform", git = "https://github.com/yanxulang/yanxu-platform.git", 修订 = "v0.6.0", 版 = "^0.6" }

[权限]
图形界面 = true
原生扩展 = true
剪贴板 = true
文件对话框 = true
```

随后运行`yanbao 更`与`yanbao 查`，让言包选择当前目标的原生制品并验证摘要与大小。

## 从 0.5.0 升级

只使用既有平台能力的应用更新依赖修订、次版本约束和锁文件即可，无需修改帧提交代码。
需要接入言界语义树时，为每个控件分配窗口内稳定正整数编号，生成协议 1.0 节点并在布局、
状态或焦点变化后更新窗口。应利用`变化`字段监控重复提交，并在关闭窗口前清除树或依赖
窗口资源清理自动回收。

## 制品与校验

Release 附带`yanxu-platform-0.6.0-six-targets.tar.gz`、同名`.sha256`和完整
`yanxu-platform-0.6.0.toml`。标签与归档包含同一批六目标动态库，清单逐项固定 SHA-256
和字节数；Release 只复用标签提交已经通过的 CI 候选。

## 安全与可靠性

- 语义树验证成功前不替换旧状态；相同树不增加修订或桥接工作。
- 修订限定在言序精确整数范围，耗尽返回稳定错误，不回绕或复用身份。
- 关闭窗口会扣除当前树、节点和文字量，同时保留累计计数与高水位供故障分析。
- 焦点和动作请求在入队前重新验证当前修订、节点能力、交互状态与参数。
- 原生依赖版本和许可集合没有变化，继续执行六目标依赖公告与许可门禁。

## 已知限制

- 0.6.0 尚未把语义树接入 UIA、NSAccessibility 或 AT-SPI；能力字段会如实返回假。
- 0.6.0 仍使用 CPU 二维栅格化，不提供 GPU 合成或离屏图层。
- 节点的可访问名称和描述由上层提供；言台不从绘制像素或系统字体中推断语义。
