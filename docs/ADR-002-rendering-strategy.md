# ADR-002：统一 CPU 二维渲染

- 状态：已接受
- 日期：2026-07-15

## 背景

首版更重视三平台一致性、可测试协议和快速建立完整垂直切片，而不是 GPU 峰值吞吐。若
分别采用 Direct2D、CoreGraphics 和不同 Linux 栈，上层绘制语义与像素结果难以冻结。

## 决策

言序一次生成完整绘制树，原生侧编译成 `YXDR` v1 二进制帧。`tiny-skia 0.11.4` 在 CPU
上实现几何、裁剪、变换、路径、阴影和混合；`cosmic-text 0.19.0` 负责复杂文字；
`softbuffer` 把最终 XRGB 整帧呈现到原生窗口。

结构化命令只作为可读入口，不用于每帧逐条跨 ABI。二进制协议具有主/次版本、命令长度、
零填充和硬上限。所有公开坐标为逻辑像素，DPI 缩放在渲染器入口统一应用。

## 后果

- 几何和混合算法在六目标一致，可在无窗口测试中断言关键像素。
- CPU 成本与整帧上传限制大型动画；能力查询明确报告`CPU二维绘制`。
- 系统字体文件仍会影响字形像素；跨平台金图需要随应用携带固定字体。
- 当前透明图层是状态透明度，不是隔离离屏组；任意旋转文字不在 0.1.0 保证内。

## 放弃方案

- 三套系统 2D API：会扩大协议差异和测试矩阵。
- 首版直接使用 wgpu：目标覆盖良好，但设备初始化、着色器、驱动差异和发布体积不符合
  最小稳定平台原语的优先级。
- 逐操作 ABI：实现简单但鼠标交互和复杂控件会产生不可接受的边界调用量。
