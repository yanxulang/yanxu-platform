# 言台 0.9.0

言台 0.9.0 不扩展公开接口，而是把 1.0 前的稳定性、性能和供应链要求变成可重复的发布
门禁。原生 ABI、四套协议和言序 API 快照均与 0.8.0 相同，现有消费者无需迁移。

## 畸形输入门禁

- 绘制协议加入 16 类版本化畸形语料，分别固定截断、魔数、主版本、命令数量、负载长度、
  非零填充和尾部数据等错误分类。
- 每个合法基准帧的全部严格前缀都必须被拒绝；逐字节写入 `0x00`/`0xff` 的边界变异只能
  得到成功解码或已定义错误，不能恐慌或越过协议上限。
- ABI v2 值覆盖未知类型、空的非零长度指针、长度溢出、无效 UTF-8、非法与重复映射键、
  过深嵌套和自引用容器。所有失败都保持稳定代码并安全释放已验证部分。

## 长序列浸泡

事件队列执行 4096 个确定性周期，累计接受 462848 个输入、合并 446464 个连续事件、拒绝
4096 个原子溢出批次并按序冲刷 16384 个事件。门禁同时核对离散事件屏障、失败批次不改写
旧队列、高水位、序号和全部累计指标。

资源生命周期执行 2048 个严格配额周期，每轮创建 15 个应用/窗口/计时器/图片/字体资源，
运行中关闭四个资源后立即创建同类替代项。共创建并关闭 30720 个资源，交替完成 1024 次
正常退出和 1024 次失败退出；帧提交/替换、无障碍树更新和原生桥激活也在关闭路径中被
回收。每轮结束都证明资源数量、图片/字体/帧字节、无障碍节点/文字、待呈现帧和活动桥为
零，最终创建与关闭账本完全相等。

## Release 性能预算

固定 Linux x86-64 作业以 Cargo `release` 模式执行四项预算：

| 场景 | 单位 | 中位数上限 |
| --- | --- | ---: |
| 绘制协议解码 | 单条命令 | 200 ns |
| 事件合并并冲刷 | 单个输入事件 | 1,000 ns |
| 资源创建、运行、退出与关闭 | 单个资源 | 2,000 ns |
| 65 条命令的 256×256 CPU 栅格帧 | 单帧 | 750,000 ns |

每项先自动校准到约 200 ms 的长样本，再丢弃两轮预热并记录九轮正式测量。判定使用单位
耗时中位数，同时要求中位绝对偏差不超过 20%；不会用六种不同硬件上的一次短计时决定
回归。版本化 JSON 报告随候选保存并进入 Release 附件。

## 零例外供应链

RustSec 已把 `ttf-parser` 标记为停止维护。0.9.0 不再沿用公告忽略，而是固定到 `fontdb`
上游已合并、获批准的 PR #92 合并提交，只保留字体发现所需的名称、语言和 OS/2 解析，
从依赖图移除 `ttf-parser`。来源策略只允许该仓库且强制完整 `rev`；上游发布包含修复的
正式包后切回 crates.io。

`cargo audit --deny warnings` 和 `cargo deny check advisories licenses sources` 均无忽略项。
Wayland 的 `quick-xml` 修复继续由仓库内固定补丁提供，不回退到受影响版本。

## 兼容与验证

- ABI v2、平台协议 1.7、事件协议 1.3、无障碍协议 1.0、绘制协议 1.1 均未改变。
- `api/api-v1.json` 与生成的 `docs/API.md` 保持一致；最低言序版本仍为 1.1.7。
- Rust 门禁共 106 项：97 项单元测试、4 项 ABI 语料、3 项绘制语料和 2 项浸泡测试。
- 六个正式桌面目标继续执行格式、测试、Clippy、Release 构建、ABI 导出、言序集成和真实
  窗口/原生无障碍桥自动退出；候选汇总还必须等待依赖策略和性能预算成功。

## 安装

Git 依赖可固定本标签：

```toml
[依赖]
言台 = { 包 = "yanxu-platform", git = "https://github.com/yanxulang/yanxu-platform.git", 修订 = "v0.9.0", 版 = "^0.9" }
```

最低言序版本仍为 1.1.7，应用仍需授予`图形界面`和`原生扩展`权限。

## 发布制品

Release 附带`yanxu-platform-0.9.0-six-targets.tar.gz`、独立 SHA-256 文件、完整
`yanxu-platform-0.9.0.toml`和`yanxu-platform-0.9.0-performance.json`。归档与标签 Git
包包含相同的 Windows、macOS、Linux x86-64/ARM64 动态库；清单逐项固定 SHA-256 与大小，
来源记录指向生成它们的成功 CI，性能报告记录同一候选的固定运行器测量。

本版本仍是桌面 CPU 二维后端，不承诺 GPU、移动端或 Web。完整方法和边界见
[性能预算](PERFORMANCE_BUDGET.md)、[兼容政策](COMPATIBILITY.md)、[安全架构](SECURITY.md)
与[第三方依赖](THIRD_PARTY.md)。
