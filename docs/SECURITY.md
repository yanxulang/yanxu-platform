# 安全架构

漏洞报告渠道与支持版本见仓库根目录 [SECURITY.md](../SECURITY.md)。本文件记录 0.6.0
实现中的技术边界。

## 输入面

| 输入 | 防护 |
| --- | --- |
| ABI v2 值 | 结构大小/标签校验、递归释放、空指针与非有限数拒绝 |
| 原生资源 | 类型、关闭状态、事件循环编号、所有者线程令牌校验 |
| 绘制帧 | 魔数/版本/长度/零填充/UTF-8 校验，16 MiB 总上限 |
| 图片 | 格式探测、16384 单边、256 MiB 解码分配上限 |
| 剪贴板文字 | 读写均验证 UTF-8 字节数，16 MiB 上限 |
| 剪贴板图片 | 仅 RGBA8；16384 单边、256 MiB 总量并严格验证`宽 × 高 × 4` |
| 字体 | 64 MiB 上限，字体数据库解析成功后才创建资源 |
| 事件 | 4096 有界队列；离散溢出为致命错误而非静默丢弃 |
| 待呈现帧 | 每窗口单槽；高频提交只替换旧帧，不累积无界帧队列；帧序号不回绕 |
| 无障碍树 | 节点、深度、文字、子与动作硬上限；编号、角色、状态、焦点和参数完整校验 |
| 密码语义 | 密码框值必须为空；敏感设置参数不进入树快照或诊断计数 |
| 文件路径 | 只来自应用显式配置、拖放或用户选择，不执行路径内容 |

FFI 调度器用 `catch_unwind` 包围 Rust 实现。错误只返回静态代码和消息，不把 Rust 对象、
回溯或平台地址暴露给言序。

共享模型、文字服务和事件循环代理使用统一恢复锁。若某个已隔离恐慌留下中毒标记，下一次
访问会记录恢复、清除标记并继续持有受 Rust 所有权保护的状态；恢复次数可由调试快照读取。
ABI 调度还会在每次调用前清空输出与错误槽，失败路径不会暴露陈旧资源值。

## 权限与同进程风险

言台动态库与宿主同进程，`原生扩展`权限意味着它不是沙箱。应用创建、剪贴板和文件对话框
仍分别检查`图形界面`、`剪贴板`和`文件对话框`权限，以限制无关系统能力。剪贴板数据在
进入系统库前验证，系统返回的数据在进入 ABI 前再次验证。拖放只把路径作为事件交给应用；
言台不会自动读取文件。

## 内存与资源

所有系统对象由 Rust 所有权管理。应用形成唯一资源根，父关闭使子状态立即失效；原子
清理标志阻止双重释放。回调 retain/release 成对，窗口表面先于窗口字段析构。图片绘制在
使用时重新解析资源句柄，已关闭句柄不会被当成像素指针。

## 供应链例外

依赖策略默认拒绝公告、未知来源和不兼容许可证。当前唯一例外是 `ttf-parser 0.25.1` 的
停止维护公告，它由 `fontdb` 间接引入，是无 unsafe 的解析库且没有已知漏洞；例外固定在
`deny.toml` 并要求每次发布复核。Wayland 构建链的 `quick-xml` 高危公告没有被忽略，而是
通过仓库内固定 `wayland-scanner 0.31.10` 源码仅升级 XML 解析器修复。完整依据见
[THIRD_PARTY.md](THIRD_PARTY.md)。

## 发布完整性

每个目标在其真实 GitHub 执行器构建并核对 ABI 导出。六目标候选包只有在所有测试和依赖
审计完成后汇总；Release 工作流验证标签 SHA 与成功 CI SHA 完全相同，再校验候选归档
摘要。发布过程不接受本地替换的二进制。

版本标签中的`dist/`与完整清单取自成功六目标候选，使 Git 包解析也受相同的文件摘要和
大小检查保护；运行和逐文件来源见[制品来源记录](ARTIFACT_PROVENANCE.md)。发布提交不得
混入本地单目标构建产生的动态库。
