# 线程模型

言台 1.0.0 采用“单一 GUI 事件循环线程 + 可锁定无句柄模型”。它不承诺窗口资源可跨
言序线程自由调用。

## GUI 线程

`应用.运行（）`在调用线程创建并运行 `winit` 事件循环，直到调用`应用.退出（）`或发生
致命后端错误。Windows 和 macOS 的窗口系统要求该调用位于进程 GUI/主线程；Linux 也应
使用同一规则，以保持六目标代码一致。

实际的 `Window`、`softbuffer::Surface`、AccessKit 适配器、渲染器和事件循环代理只由
`Runner` 持有并在该线程访问。言序可以在运行前创建窗口模型；系统窗口会在事件循环
`resumed` 后建立。

## ABI 身份

每个 ABI v2 调用带`event_loop_id`和`owner_thread_token`。原生资源在创建时保存这两个值；
后续解析句柄时必须完全一致，否则返回`PLATFORM_WRONG_THREAD`。这同时阻止把一个宿主
或应用的句柄传给另一个宿主。

高层 API 没有把资源声明为可发送对象。后台工作应只计算普通言序数据或构造命令树，
再通过宿主已有的消息机制回到所有者线程提交窗口状态。

## 共享模型

应用模型和文字服务分别放在 `Arc<Mutex<_>>` 中，锁只保护短操作：读取/更新资源、取出
事件批次、计算计时器截止点或执行一次文字服务调用。系统对话框、事件循环等待和像素
呈现不会持有模型锁。

若一次已被 FFI 隔离的恐慌使锁中毒，统一锁入口会累计一次恢复并清除中毒标记。恢复不
改变线程所有权规则，也不重试发生恐慌的原操作；它只保证后续诊断、退出与析构不会因
标准库中毒标记再次恐慌。

## 原生无障碍并发

UIA、NSAccessibility 或 AT-SPI 可能从平台选择的线程请求首次语义树。每个窗口因此保存
一个只含标题、物理尺寸、比例因子和不可变语义树引用的线程安全快照；激活处理器只锁定
该快照并返回完整树，不访问 `Window`、表面或渲染器。窗口必须先隐藏创建，注册适配器后
才按模型设置可见，避免辅助技术在桥存在前观察到窗口。

系统动作与停用通知通过带类型的 `EventLoopProxy` 返回 GUI 线程。动作转换和当前树校验
在同一次模型锁持有期间完成，然后进入既有有界事件队列；上层回调仍不会与同一窗口事件
并发。树同步先替换快照并释放快照锁，再调用活动适配器，避免模型锁与平台回调形成反向
锁顺序。密码值不进入快照。

事件循环同步顺序为：

```text
接收系统事件 → 更新模型/排队 → 批量回调 → 同步言序产生的新状态 → 呈现 → 等待
```

回调由 ABI v2 `callback_post` 投递，随后调用宿主 `pump(4096)`。回调执行在事件循环的
投递点，不与同一应用的窗口事件并发。回调可以提交帧、创建/关闭资源或请求退出；这些
变化在下一次模型同步生效。

每个窗口只保存一个待呈现帧。提交发生在旧帧呈现前时，新帧替换旧帧并累计背压指标；
内部代次保证较慢的旧帧呈现不会清除较新的待呈现状态。提交还会唤醒 `winit` 与宿主，
因此后台消息回到所有者线程后不必等待下一次系统输入才触发同步。

成功提交原生表面后，事件循环排入带帧序号和单调时间的`帧呈现`事件，并在下一投递点
冲刷。连续动画在该回调中提交下一帧，因此不会与当前呈现并发，也不需要另建渲染线程。
窗口不可呈现时反馈自然暂停；恢复重绘后继续。

## 唤醒

运行开始时，后端按 `event_loop_id` 注册一个可区分普通唤醒与无障碍事件的
`EventLoopProxy`，结束时通过守卫注销。
`应用.唤醒（）`先发送用户事件唤醒 `winit`，再调用宿主 wake 钩子。若循环尚未运行或已
结束，代理查找失败是无害的；调用本身仍完成。

计时器不会创建工作线程。事件循环把控制流设为最早截止时间的 `WaitUntil`，醒来后批量
产生计时器事件。

## 阻塞操作

文件对话框和系统剪贴板 API 是同步平台操作，应在 GUI 所有者线程调用。文件解码、字体
整形和 CPU 渲染也在调用/事件循环线程完成；大输入受硬上限和应用剩余配额保护。1.0.0
尚未提供内建线程池或异步渲染，应用应避免在事件回调中做无界 I/O。

## 退出与清理

首次运行先原子进入运行状态并永久冻结资源配额；同一应用不能并发运行，也不能在退出后
重启。退出请求只在首次状态变化时唤醒事件循环，后续请求幂等。循环守卫保证无论事件循环
创建失败、表面创建失败、正常返回还是致命后端错误，活动标记都清除、状态进入已退出并
记录稳定错误；事件循环代理同时注销。关闭根资源直接进入已关闭，守卫不会重新生成节点。
窗口按“表面、无障碍适配器、窗口”的字段析构顺序释放；模型关闭活动窗口时同步扣除原生
桥当前量。资源所有权的完整规则见
[RESOURCE_LIFETIME.md](RESOURCE_LIFETIME.md)。
