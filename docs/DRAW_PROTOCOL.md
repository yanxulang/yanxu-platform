# 绘制协议 v1.1

言台用一个版本化二进制帧提交整次绘制。结构化命令只是言序侧易用入口；`绘制编码`
会先把整列命令确定性编译成本文格式，窗口随后一次跨越 ABI 提交该缓冲。

## 标量与字符串

所有整数与 IEEE 754 `f32` 都是小端序。浮点数必须有限。`RGBA` 是四个连续 `u8`，使用
非预乘 0..255 通道；渲染器内部再转换为预乘颜色。`text` 的编码为`u32 字节数`加 UTF-8
字节，不含终止符。

矩形统一写作四个 `f32`：`x, y, width, height`。公开坐标是逻辑像素，呈现时由窗口比例
因子统一缩放。

## 帧

| 偏移 | 大小 | 字段 |
| ---: | ---: | --- |
| 0 | 4 | ASCII 魔数 `YXDR` |
| 4 | 2 | 主版本，当前为 1 |
| 6 | 2 | 次版本，当前为 1 |
| 8 | 4 | 帧标志，当前必须由发送方写 0 |
| 12 | 4 | 命令数量 |

每条命令紧随其后：

| 大小 | 字段 |
| ---: | --- |
| 2 | 操作码 |
| 2 | 命令标志，当前为 0 |
| 4 | 负载字节数 |
| 变长 | 负载 |
| 0..3 | 补齐到四字节边界的零字节 |

非零填充、截断、尾部多余数据、错误魔数和不同主版本都会被拒绝。同一主版本的未知操作
码可以按负载长度跳过，这是次版本向前兼容的基础。

## 操作码

下表中的 `rect`、`RGBA` 和 `text` 使用上面的复合编码。

| 码 | 结构化类型 | 负载 |
| ---: | --- | --- |
| 1 | `清空` | `RGBA` |
| 2 | `保存` | 空 |
| 3 | `恢复` | 空 |
| 4 | `裁剪矩形` | `rect` |
| 5 | `变换` | 六个 `f32`：a,b,c,d,e,f |
| 6 | `图层` | `f32 opacity` |
| 7 | `填充矩形` | `rect, RGBA` |
| 8 | `描边矩形` | `rect, f32 width, RGBA` |
| 9 | `圆角矩形` | `rect, f32 radius, RGBA` |
| 10 | `直线` | `f32 x1,y1,x2,y2,width, RGBA` |
| 11 | `圆形` | `f32 x,y,radius, RGBA` |
| 12 | `阴影` | `rect, f32 radius,blur,offset_x,offset_y, RGBA` |
| 13 | `文字` | `f32 x,y,max_width,font_size,line_height, RGBA, text` |
| 14 | `图片` | `u64 resource, rect, f32 opacity` |
| 15 | `路径` | 见下文 |
| 16 | 带样式`文字` | 文字公共字段、样式与两个 `text` |
| 17 | `字形序列` | 见下文 |

操作码 1..15 属于 v1.0；16..17 在 v1.1 增加。结构化`文字`不含`字族`、`字重`或`斜体`
时仍编码为 13；出现任一字体样式字段时编码为 16。

## 状态

初始变换是窗口 DPI 比例，初始裁剪为空，初始透明度为 1。`保存`复制变换、裁剪和透明度，
`恢复`弹栈；帧结束时栈必须为空，深度上限 256。`裁剪矩形`与现有裁剪取交集。`变换`
把 2D 仿射矩阵后乘到当前状态。

`图层`把当前透明度乘以给定的 0..1 值；通常写成`保存 → 图层 → 绘制 → 恢复`。0.1.0
没有隔离的离屏组，因此组内重叠像素仍按每个原语分别混合。

## 路径

路径负载是：

```text
u8 style          0=填充，1=描边
u8 even_odd       0=非零环绕，1=奇偶
u16 verb_count
f32 stroke_width  填充时仍占位
RGBA color
verb[]
```

每个 verb 先写一个 `u8`：1 移动（2 个 f32）、2 直线（2 个）、3 二次曲线（4 个）、
4 三次曲线（6 个）、5 闭合（无坐标）。描边宽度必须为正。

## 样式文字

操作码 16 的负载是：

```text
f32 x, y, max_width, font_size, line_height
RGBA color
u16 weight        1..1000
u8 italic         0 或 1
u8 reserved       必须为 0
text family       空串表示系统无衬线
text content
```

渲染器用 `cosmic-text` 做高级整形和字体回退。当前文字/字形绘制对平移与缩放给出一致结果；
任意旋转的字形不会转换成矢量轮廓，因而不作为 0.1.0 保证。

## 字形序列

操作码 17 重放已经整形的字形，避免渲染阶段再次改变光标布局：

```text
f32 origin_x, origin_y, font_size
RGBA color
u16 weight
u8 italic
u8 reserved       必须为 0
text family
u32 glyph_count
glyph[glyph_count]

glyph = u16 glyph_id, u16 reserved=0, f32 x, f32 baseline
```

一个序列共享字族、字重、斜体、字号和颜色。发生字体回退时，应按整形结果的实际`字族`、
`字重`与`斜体`分组为多个序列。`origin`加上字形相对位置后再应用当前变换。

## 上限与错误

| 项目 | 上限 |
| --- | ---: |
| 整帧 | 16 MiB |
| 单命令负载 | 4 MiB |
| 命令数 | 65536 |
| 单文字字段 | 4 MiB UTF-8 |
| 字形序列 | 262144 个 |
| 状态栈 | 256 层 |
| 表面单边 | 16384 物理像素 |
| 图片单边 | 16384 像素 |

主版本不符返回`PLATFORM_DRAW_MAJOR`；超限、UTF-8、非有限数字和其他损坏分别映射到
`PLATFORM_DRAW_LIMIT`、`PLATFORM_DRAW_UTF8`、`PLATFORM_DRAW_NUMBER`与
`PLATFORM_DRAW_CORRUPT`。

## 兼容与确定性

- 主版本只在无法安全跳过或解释旧字段时提升。
- 次版本可以增加操作码；旧接收方跳过未知码，新接收方继续支持旧码。
- 既有操作码的字段顺序、单位和范围在同一主版本内不改变。
- CPU 渲染、固定依赖版本和小端协议使六目标走同一算法；字体像素仍受各系统实际字体
  文件影响，因此跨系统像素金图必须附带同一自定义字体。
