# 言台 0.5.0

言台 0.5.0 把整帧提交升级为可追踪的呈现反馈链。应用既可继续使用原有返回命令数的
提交方法，也可取得帧序号和替换回执，并在帧成功交给原生表面后收到单调时间事件。

## 新功能

- 新增`窗口.提交帧反馈（命令列）`与`窗口.提交缓冲反馈（字节串）`；回执包含`帧序号`、
  `命令数`、`替换`、`被替换帧`和`提交时间`。
- 事件协议提升到 1.2，新增离散的`帧呈现`事件，报告帧序号、提交时间、呈现时间和延迟
  毫秒。
- 窗口查询新增最后帧序号、待呈现帧序号和帧提交时间；尚未提交或没有待呈现帧时使用
  `空`表达。
- 平台协议提升到 1.4，能力查询明确报告提交反馈、呈现反馈、单调时间基准和动画驱动
  事件。

## 动画与背压

连续动画在首次`需要重绘`时提交一帧，随后由匹配的`帧呈现`事件驱动下一帧。窗口最小化、
遮挡或表面尺寸为零时反馈可以暂停，恢复重绘后继续，不需要持续运行高频计时器。

每个窗口仍只有一个待呈现槽。新回执会指出是否替换了旧帧以及被替换的帧序号；被替换帧
不会再产生呈现反馈。帧序号限定在言序可精确表达的正整数范围内，耗尽返回
`PLATFORM_FRAME_SEQUENCE`，不会回绕或复用身份。

## 兼容性与平台

- 原生 ABI 仍为 v2，绘制协议仍为 1.1；平台协议从 1.3 提升到 1.4，事件协议从 1.1
  提升到 1.2。
- 既有`提交帧`和`提交缓冲`签名与返回值不变；旧消费者忽略未知事件和查询字段即可继续
  工作。
- 最低言序仍为 1.1.7，继续覆盖 Windows、macOS、Linux 的 x86_64 与 ARM64 六个目标。

## 安装示例

```toml
[依赖]
言台 = { 包 = "yanxu-platform", git = "https://github.com/yanxulang/yanxu-platform.git", 修订 = "v0.5.0", 版 = "^0.5" }

[权限]
图形界面 = true
原生扩展 = true
剪贴板 = true
文件对话框 = true
```

随后运行`yanbao 更`与`yanbao 查`，让言包选择当前目标的原生制品并验证摘要与大小。

## 升级方式

从 0.4.0 升级只需更新依赖修订、次版本约束和锁文件。无需改动现有整帧提交代码。需要
动画反馈时先检查平台次版本或能力字段，再改用反馈提交方法，并忽略不匹配当前等待帧的
呈现事件。

## 制品与校验

Release 附带`yanxu-platform-0.5.0-six-targets.tar.gz`、同名`.sha256`和完整
`yanxu-platform-0.5.0.toml`。标签与归档包含同一批六目标动态库，清单逐项固定 SHA-256
和字节数；Release 只复用标签提交已经通过的 CI 候选。

## 安全与可靠性

- 帧、绘制缓冲、事件队列和资源仍受原有硬上限与单槽背压约束。
- 呈现事件使用进程内单调时间，不接受或生成墙上时间，不受系统时钟调整影响。
- 真实窗口测试必须收到与提交回执匹配的呈现事件；等待超时会使门禁失败。

## 已知限制

- `帧呈现`表示软件缓冲已成功交给原生表面，不表示显示器已完成扫描，也不是垂直同步
  时间戳。
- 0.5.0 仍使用 CPU 二维栅格化，不提供 GPU 合成或离屏图层。
- 无头环境只能验证模型和错误路径；Linux 虚拟显示门禁会运行真实窗口并等待呈现反馈。
