# 文字与中文输入法

言台把“按键”“输入文字”“IME 组合”和“文字布局”分开。编辑器应以文字整形返回的 UTF-8
索引和字形位置维护光标，不应假设一个 Unicode 标量对应一个字形。

## 字体来源

`应用.字体族（）`枚举当前系统数据库中的去重字族。`匹配字体`接受`字族`、`字重`
（1..1000）和`斜体`，返回实际字族、PostScript 名称、实际字重、斜体与等宽标志；找不到
时返回空。

`应用.加载字体（内容）`接受最大 64 MiB 的字体字节。只有解析后新增了至少一个字族才
创建字体资源。该字体同时加入应用文字服务，并在窗口下次绘制前同步到渲染器；因此整形
与字形重放能看到同一文件。关闭字体资源会解除模型所有权，但已创建渲染器中的字体缓存
在本次应用生命周期内不会主动卸载。

## 整形配置

```yanxu
定 布局：典 为 应用.文字整形（「A中ب」，{
    「字族」：「Noto Sans CJK SC」，
    「字重」：500，
    「斜体」：假，
    「字号」：16，
    「行高」：22，
    「最大宽」：320，
    「换行」：真
}）；
```

省略字族时使用系统无衬线并允许字体回退。`cosmic-text` 执行脚本识别、双向文本、连字、
字距和回退；换行策略为词或字形边界。文字最大 4 MiB，字号、行高与最大宽必须为有限
正数且不超过 1,000,000。

布局顶层返回`宽`、`高`、首行`基线`、`行`和`字形`。每行包含：

- `原文行`、`顶部`、`基线`、`行高`、`宽`、`从右至左`；
- 半开字形范围`字形起..字形终`。

每个字形包含：

- 实际回退后的`字族`（兼容别名`字体`）、`字重`与`斜体`；
- `字形`编号、`横坐标`、`基线`与点击宽度；
- 半开原文范围`原文起..原文终`，单位是 UTF-8 字节；
- `从右至左`方向。

一个字形可能覆盖多个字符，多个字形也可能指向同一原文簇。所有编辑操作都应把返回值
限制在原文的 UTF-8 边界。

## 测量、命中与绘制

`文字测量`只返回宽、高、基线、行高与行数。`文字命中（内容，配置，x，y）`选择对应行，
再按字形中点和方向返回原文字节位置；超出上方/下方时使用首行/末行，空布局返回 0。

简单标签可直接提交`文字`命令。需要可重复光标定位时，先整形，再按相邻字形的实际字族、
字重和斜体分组，提交`字形序列`：

```yanxu
定 首：典 为 布局【「字形」】【0】；
窗口.提交帧（【
  {「类型」：「字形序列」，「位置」：【20，20】，「字号」：16，
   「颜色」：【20，24，32，255】，
   「字族」：首【「字族」】，「字重」：首【「字重」】，「斜体」：首【「斜体」】，
   「字形」：【
      {「字形」：首【「字形」】，
       「横坐标」：首【「横坐标」】，「基线」：首【「基线」】}
   】}
】）；
```

字形编号只在对应字体面内有意义，不能与另一字族混用。若必须跨机器像素一致，应把字体
作为受许可的应用资源一起发布，而不是依赖系统字体集合。

## 输入法配置

文本控件获得编辑焦点时调用：

```yanxu
窗口.配置输入法（真，【光标横，光标纵，1，行高】，「普通」）；
```

光标区域使用窗口逻辑像素，系统用它放置中文候选窗。密码框使用`密码`，终端使用`终端`；
其余值按`普通`处理。失焦或控件变为只读时调用`停止输入法（）`。

## 组合事件

```text
IME组合开始
IME组合更新 {文本, 选区起?, 选区终?}
IME组合提交 {文本}
IME组合取消
```

`IME组合更新`是尚未写入文档的预编辑文本；更新事件应替换上一段预编辑，而不是追加。
可选选区是预编辑文本内的 UTF-8 字节范围。`IME组合提交`才进入撤销历史和正式文档；取消
只清除预编辑状态。提交事件不应再次通过按键事件插入。

按键事件中的`文本`是系统对该键的即时解释，独立的`文本输入`用于非 IME 可插入文本。
组合期间应优先消费 IME 生命周期，避免重复插入。

## 已验证范围

原生单元测试用中文预编辑`言序`验证开始、选区、提交和取消映射；言序 ABI 集成测试验证
中文整形、基线、回退字族以及整形字形可重新编码为字形序列。Linux CI 的真实窗口在 Xvfb
运行；实际候选窗和操作系统输入法引擎仍需要目标桌面的人工冒烟，因为 CI 没有用户输入法
会话。此限制不改变事件和候选区定位协议。
