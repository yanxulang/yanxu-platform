# 言台 0.7.0

言台 0.7.0 把 0.6.0 冻结的有界语义树正式接入 Windows UIA、macOS
NSAccessibility 与 Linux AT-SPI。三个系统后端共享同一角色、状态、动作、文字和安全
校验实现；言界不需要维护三套平台语义分支。

## 主要变化

- 平台协议从 1.5 提升到 1.6；无障碍协议保持 1.0，事件协议保持 1.3，原生 ABI 保持 v2。
- `原生无障碍桥`在六个正式目标为真；`原生无障碍后端`按目标返回`UIA`、
  `NSAccessibility`或`AT-SPI`。
- 每个系统窗口先隐藏创建，在首次显示前注册原生适配器，再按模型设置可见状态。
- 使用稳定编号 0 作为原生窗口根；协议正整数节点编号和合成文字运行编号位于互不冲突的
  命名空间。
- 映射全部 38 个协议角色、18 类状态和 15 个动作；名称、描述、值、逻辑边界、子关系与
  当前焦点进入同一完整原生树。
- 语义树修订、标题、客户区物理尺寸或 DPI 变化时，活动适配器同步最新完整树；所有
  `winit`窗口事件先交给原生适配器。
- 输入框和多行输入框生成稳定的原生文字运行。Unicode 字素长度保留 UTF-8 字节数，
  UIA、NSAccessibility 或 AT-SPI 选择范围可还原为事件协议要求的字节边界。
- 系统焦点、点击/选择、设置值、展开/折叠、增减、滚动、滚动到、上下文菜单以及自定义
  操作通过事件循环代理回到现有焦点/动作事件。
- 调试快照增加原生桥当前激活、高水位、激活/停用、树同步、请求和原生拒绝总数。

## 原生后端

| 系统 | x86-64 | ARM64 | 系统接口 |
| --- | --- | --- | --- |
| Windows | 支持 | 支持 | UI Automation |
| macOS | 支持 | 支持 | NSAccessibility |
| Linux | 支持 | 支持 | AT-SPI（Wayland / X11 窗口） |

窗口、事件循环和无障碍桥仍只由 GUI 线程持有。平台可能从其他线程请求首次语义树，因此
激活处理器只读取线程安全、无平台句柄的不可变快照；系统动作与停用通知统一返回 GUI
事件循环。应用代码和言序 ABI 不会接触 UIA、AppKit、AT-SPI 或 AccessKit 对象。

## 安全与隐私

- 原生动作收到后，在同一模型锁内重新查询当前树和节点，再调用 1.0 协议既有校验。
- 已删除、隐藏、禁用、未声明动作、参数畸形、选区越界或命名空间不匹配的请求被拒绝，
  不会直接修改上层控件状态。
- 离散请求仍受 4096 事件队列上限保护；队列耗尽保持致命错误，不静默丢失辅助技术操作。
- 文本选择以 Unicode 字素为原生字符单位，并无损转换为 UTF-8 字节边界；单字段仍受
  65536 字节上限保护。
- 密码框继续要求语义值为空，不生成原生文字运行，也不向系统桥声明密码文字选区。
  辅助技术设置的新口令只经过一次有界动作参数，不进入快照、日志或诊断值。
- 原生适配器先于系统窗口析构；关闭活动窗口会扣除当前桥数量并保留累计诊断。

## 兼容性

0.6.0 的语义树、动作参数、焦点/动作事件、错误代码和硬上限保持不变。仅使用 1.5 能力的
消费者可以忽略 1.6 新字段。需要系统辅助技术的消费者应同时确认：

```yanxu
定 协议：典 为 平台.协议查询（）；
定 能力：典 为 平台.能力查询（）；
# 协议【「平台主」】 = 1，协议【「平台次」】 >= 6
# 协议【「无障碍主」】 = 1
# 能力【「无障碍语义树」】 = 真
# 能力【「原生无障碍桥」】 = 真
# 能力【「原生无障碍后端」】为 UIA、NSAccessibility 或 AT-SPI
```

平台次版本本身不是非桌面目标的能力证明；仍须读取能力字段。无障碍主版本不同时必须
拒绝组合。

## 安装与升级

言包 Git 依赖示例：

```toml
[依赖]
言台 = { 包 = "yanxu-platform", git = "https://github.com/yanxulang/yanxu-platform.git", 修订 = "v0.7.0", 版 = "^0.7" }

[权限]
图形界面 = true
原生扩展 = true
```

从 0.6 升级时更新修订、次版本约束和锁文件即可；公共言序方法签名没有变化。上层应在
窗口显示前或首次帧前提交语义树，并在处理焦点/动作事件后提交反映最终状态的新快照。

## 验证

- 81 项 Rust 单元测试覆盖角色、状态、动作、树边界、线程安全激活、字素选区、动作消歧、
  滚动单位、模型重校验、桥生命周期、拒绝诊断和密码保护。
- 六个正式目标分别执行格式、单测、Clippy、Release 构建、ABI v2 导出、依赖安全与许可、
  言序 1.1.7 集成以及提交语义树的真实窗口自动退出冒烟；Linux 使用 Xvfb。
- Release 附带`yanxu-platform-0.7.0-six-targets.tar.gz`、同名`.sha256`和完整
  `yanxu-platform-0.7.0.toml`。标签与归档包含同一批六目标动态库，清单逐项固定 SHA-256
  与字节数。

## 已知边界

- 0.7.0 只承诺六个桌面目标，不提供移动端或 Web 无障碍桥。
- 屏幕阅读器的最终措辞、快捷键和激活时机由系统、语言、桌面会话与辅助技术决定；无
  辅助技术的自动化环境验证适配器、树同步和窗口生命周期，不冒充人工播报验收。
- 密码设置受支持，但密码文字选区不通过原生桥暴露。
- CPU 二维栅格化保持不变；本版本不提供 GPU 合成、离屏图层或原生控件包装。
