# 原生 ABI v2 冻结

言台 1.0 的动态库入口固定为`yanxu_native_module_v2`，只面向六个 64 位小端目标。该
边界以言序 1.1.7 的规范 C 头文件和提交`7d0e8b2b7bdd1125a7d271bce690ab854c79c31c`
为最低宿主基线。完整机器契约位于
[`api/native-abi-v2.json`](../api/native-abi-v2.json)，包含：

- 状态值、值类型标签与标志位；
- C 调用约定、句柄别名和 11 个函数指针签名；
- 所有 C 布局结构的大小、对齐和字段偏移；
- 模块名称、能力位、释放函数状态；
- 41 个原生操作的稳定编号与名称；
- 5 个资源类型的稳定顺序与名称。

六目标测试从实际 Rust `repr(C)` 类型和导出的模块描述符重新生成同一 JSON 并逐字节
比较，同时用精确 Rust 类型断言核对全部 C 函数指针。测试读取真实导出的模块名、结构
大小、操作名称/编号/调用指针和资源类型，而不以源码静态表替代描述符。操作只能在表尾
追加；重排、删除、复用编号或导出空调用指针都会使门禁失败。资源类型名称、值标签、
字段类型、偏移、大小、调用约定或入口符号的任何不兼容变化都必须使用新的 ABI 入口和
ABI 版本，不能在 1.x 中静默修改本快照。

`struct_size`允许未来宿主或模块在相同 ABI 主版本中只向结构尾部追加可忽略字段；读取方
必须先检查调用方提供的大小。现有字段不能移动或改变含义。字符串和字节均由指针加显式
长度表达，不要求 NUL 结尾；非空长度的空指针必须拒绝。

言序层公开声明另由 [`api/api-v1.json`](../api/api-v1.json) 冻结。两份快照分别保护语言
API 与进程内原生边界，任一通过都不能替代另一份。
