# 变更日志

本项目遵循语义化版本。

## [1.0.0] - 2026-07-21

### 稳定接口

- 将言序公开 API、原生 ABI v2 的 C 布局、11 个函数签名、41 个操作和 5 个资源类型
  固定为机器可比较契约；不兼容修改必须进入新的 ABI 或包主版本。
- 冻结平台 1.7、事件 1.3、无障碍 1.0 和绘制 1.1 协议，覆盖能力字段、39 个事件及合并
  方式、38 个无障碍角色、18 个状态、15 个动作、操作码和全部主要容量上限。
- 建立 70 个实际运行时错误码的版本化目录；1.x 不删除、改名或复用既有代码含义。

### 兼容与迁移

- 明确 1.x 语义化版本、弃用、支持周期、安全例外和六目标承诺；支持言序范围固定为
  `>=1.1.7, <2.0.0`。
- 新增 0.x 到 1.0 迁移指南。0.9.0 消费者只需更新依赖和锁文件，运行时接口和协议组合
  保持不变。

### 工具链与供应链

- 新增精确 Rust 1.92.0 最低工具链作业，执行格式、全目标测试和警告即错误 Clippy，
  并纳入六目标候选汇总门禁。
- 再次复核 `fontdb` 正式发布状态；修复尚未进入 crates.io，因此继续固定已经评审的
  上游提交，并保持依赖图不含停止维护的 `ttf-parser`。

### 测试

- Rust 门禁合计 109 项：100 项单元测试、4 项 ABI 畸形语料、3 项绘制协议语料和 2 项
  长序列浸泡测试；默认工具链与 Rust 1.92.0 均通过全目标 Clippy。

## [0.9.0] - 2026-07-21

### 稳定性

- 增加 16 类版本化绘制协议畸形语料、所有合法帧严格前缀截断检查和逐字节边界变异，
  要求错误分类稳定且任何输入都只能得到有界结果。
- ABI v2 值边界新增未知类型、空指针、长度溢出、无效 UTF-8、非法/重复映射键、过深
  嵌套和自引用容器门禁。
- 增加 4096 轮事件背压浸泡，验证连续事件合并、离散屏障、原子批次回滚、顺序、水位和
  累计指标。
- 增加 2048 轮严格配额资源生命周期浸泡，覆盖运行中关闭和同类重建、帧/无障碍替换、
  正常与失败退出，并证明资源、持有字节、帧和无障碍账本最终归零。

### 性能

- 新增 Release 性能预算运行器，覆盖绘制协议解码、事件合并与冲刷、资源生命周期和
  256×256 CPU 栅格帧。
- 每项先自动校准到长样本，再执行两轮预热和九轮测量；使用单位耗时中位数判定，并要求
  MAD 不超过 20%。性能只在固定 Linux x86-64 作业判定，不用六种硬件的单次短计时。
- CI 保存版本化 JSON 报告，六目标候选汇总必须等待性能、依赖策略和全部原生矩阵成功。

### 安全

- 从依赖图移除停止维护的 `ttf-parser`，固定采用 `fontdb` 上游已合并并获批准的最小解析
  修复提交；Git 来源必须使用完整 `rev`，上游正式发布后切回 crates.io。
- `cargo-audit` 与 `cargo-deny` 不再含公告忽略项；已知漏洞、不健全和停止维护警告都会
  阻止候选发布。

### 兼容

- 原生 ABI 保持 v2，平台协议保持 1.7，事件协议保持 1.3，无障碍协议保持 1.0，绘制
  协议保持 1.1；公开 API 快照没有变化，0.8.0 消费者无需迁移。

### 测试

- Rust 门禁合计 106 项：97 项单元测试、4 项 ABI 畸形语料、3 项绘制协议语料和 2 项
  长序列浸泡测试；全目标 Clippy 继续以警告即错误执行。

## [0.8.0] - 2026-07-21

### 新增

- 平台协议提升至 1.7，新增`配额查询`与`配额配置`操作，以及言序层
  `应用.资源配额（）`和`应用.配置资源配额（配置）`。
- 应用可在首个子资源或首次运行前下调资源总数、各资源数量、图片/字体持有字节、保留帧
  字节和无障碍树总量；首次使用后配额永久冻结。
- 调试快照公开配额上限、当前使用、冻结状态、按原因与类别划分的拒绝统计，以及应用
  生命周期、退出错误、运行/退出/关闭与资源归零计数。
- 应用生命周期区分就绪、运行中、退出请求、已退出和已关闭；退出请求和运行收敛均幂等，
  已退出应用不会重新启动事件循环。

### 安全

- 资源、窗口、计时器、图片、字体、帧和无障碍总量均在修改状态前使用检查运算和稳定
  `PLATFORM_QUOTA_*`错误拒绝，超限不会分配资源编号或改写旧状态。
- 初始窗口事件使用原子批次；任一入队失败会恢复编号、父子图、配额冻结、使用量和资源
  指标，不留下半创建窗口。
- 字体在修改文字服务前预检数量与持有字节；图片在解码前预检资源槽，并把解码分配限制为
  应用剩余图片字节额度。

### 兼容

- 原生 ABI 仍为 v2，事件协议仍为 1.3，无障碍协议仍为 1.0，绘制协议仍为 1.1；1.6
  消费者可忽略新增能力与诊断字段。
- 新配额默认值等于 0.7 的既有硬边界；未调用配置接口的应用保持原有可用容量。

### 测试

- Rust 单元测试增至 97 项，覆盖各配额、拒绝计数、解析单计数、事件原子性、字体/图片
  预检、生命周期单向迁移和运行错误收敛。
- 增加 128 轮混合窗口、计时器、图片、字体、帧和无障碍树关闭压力测试；累计 1920 个
  资源后实时数量、持有字节、待呈现帧和无障碍当前量全部归零，创建/关闭账本完全相等。

## [0.7.0] - 2026-07-21

### 新增

- 平台协议提升至 1.6；能力查询按目标报告 UIA、NSAccessibility 或 AT-SPI 原生桥。
- 每个窗口在首次显示前注册 AccessKit 适配器，激活时同步返回稳定窗口根与完整语义树。
- 38 个角色、18 类状态和 15 个动作转换到统一原生树；逻辑边界按实际 DPI 转换为物理像素。
- 输入框生成稳定文字运行，Unicode 字素位置可无损转换为协议要求的 UTF-8 字节选区。
- 调试快照增加原生桥当前激活、高水位、激活/停用、树同步、请求与拒绝计数。

### 安全

- 系统焦点和动作回到事件循环后，重新校验当前树、节点身份、可见/启用状态、声明动作、
  参数形状和有界队列；过期或畸形请求不会直接修改上层状态。
- 密码框不向原生树生成值或文字运行；设置的新口令只存在于一次有界动作参数中。
- 原生窗口根使用保留编号 0，协议节点仍限定为正整数，合成文字编号使用独立命名空间。

### 兼容

- 无障碍协议仍为 1.0、事件协议仍为 1.3、ABI 仍为 v2；现有树和事件字段含义不变。
- 1.5 消费者可忽略新增能力与诊断字段；需要系统辅助技术的消费者必须检查
  `原生无障碍桥`和`原生无障碍后端`。

### 测试

- 增加角色/状态/动作映射、线程安全激活、字素选区、动作消歧、滚动、桥生命周期和拒绝
  诊断测试；六个正式目标都执行真实窗口与原生桥自动退出冒烟。

## [0.6.0] - 2026-07-21

### 新增

- 平台协议提升至 1.5、事件协议提升至 1.3，并增加独立无障碍协议 1.0 协商。
- 窗口可原子更新、查询和幂等清除语义树；相同规范化快照去重，修订单调递增。
- 定义 38 个角色、18 个状态和 15 个动作的角色组合、参数与单一焦点校验。
- 增加经过当前树修订验证的`无障碍焦点请求`与`无障碍动作请求`事件模型。
- 调试快照增加语义树当前量、高水位、更新/去重/清除、请求和拒绝计数。

### 安全

- 单窗口语义树限制为 16384 节点、根为 0 时最大 64 的嵌套深度、4194304 文字字节，并
  限制单节点子与动作数。
- 重复编号、未知角色、状态/动作组合、非有限边界和畸形动作参数使用稳定错误拒绝整次更新。
- 密码框语义值必须为空，防止口令进入可查询快照或诊断路径。

### 兼容

- 新操作和事件只做次版本追加；既有窗口、帧、剪贴板和事件字段含义不变。
- `原生无障碍桥`在 0.6.0 明确为假；消费者不能把语义协议支持当作系统桥已经可用。

### 测试

- 增加树上限、角色/状态/动作、焦点、参数、队列背压、修订去重、关闭回收、指标和密码值回归测试。

## [0.5.0] - 2026-07-20

### 新增

- 平台协议提升至 1.4、事件协议提升至 1.2，并协商帧提交、呈现与动画驱动能力。
- 新增兼容的帧提交反馈方法，返回帧序号、命令数、单调提交时间和被替换帧序号。
- 成功提交原生表面后投递`帧呈现`事件，报告提交时间、呈现时间与延迟。
- 窗口查询公开最后帧、待呈现帧及提交时间，便于恢复和诊断。

### 兼容

- 既有`提交帧`和`提交缓冲`仍返回命令数；旧消费者可忽略新增事件和查询字段。
- 连续动画使用呈现反馈驱动下一帧；不可呈现窗口暂停反馈，恢复重绘后继续。

### 测试

- 增加帧序号耗尽、替换回执、呈现字段、能力协商、真实 ABI 与真实窗口反馈回归测试。

## [0.4.0] - 2026-07-20

### 新增

- 平台协议提升至 1.3，并协商剪贴板文字、图片、格式与容量上限。
- 增加 RGBA8 剪贴板图片读取和写入包装，宽、高与内容使用普通言序数据表达。

### 安全

- 剪贴板文字读写限制为 16 MiB UTF-8；系统返回的超限内容也会被拒绝。
- 剪贴板图片限制为单边 16384、总内容 256 MiB，并严格验证`宽 × 高 × 4`与字节数相等。
- 尺寸乘法使用检查运算，格式、形状和容量错误分别返回稳定代码。

### 测试

- 增加文字边界、图片格式/尺寸/容量、能力协商和言序公开包装错误路径回归测试。

## [0.3.0] - 2026-07-20

### 新增

- 平台协议提升至 1.2，并协商运行可观测性与单窗口待呈现帧上限。
- 调试快照增加事件队列、资源生命周期和帧渲染的结构化累计指标与高水位。
- 每个窗口使用单槽待呈现帧；新帧有界替换旧帧，并以内部代次避免旧呈现清除新状态。
- 帧提交会唤醒等待中的事件循环，窗口查询报告待呈现状态与当前帧字节数。

### 修复

- 滚轮增量合并改为原子更新，任一轴溢出时不会部分改写已排队事件。

### 测试

- 增加队列背压、水位、资源关闭、帧替换、旧代次呈现和言序诊断接口回归测试。

## [0.2.0] - 2026-07-20

### 新增

- 平台协议提升至 1.1，并通过能力查询报告状态故障恢复支持。
- 调试快照增加进程内状态锁恢复次数，便于生产诊断故障影响。

### 修复

- 模型、文字服务和事件循环代理的中毒锁统一恢复，避免一次恐慌导致后续调用二次崩溃。
- ABI 调度开始时清空旧输出与错误状态，失败调用不再把调用方的陈旧资源值留在输出槽。

### 测试

- 增加锁中毒故障注入和失败 ABI 输出清理回归测试。

## [0.1.0] - 2026-07-15

### 新增

- 冻结绘制命令缓冲协议 v1 的帧头、长度分帧和兼容规则。
- 增加完整统一事件枚举、批处理、连续事件合并、滚轮累积和有界队列。
- 增加应用、窗口、定时器、图片和字体的父子资源所有权模型。
- 接入言序 1.1.7 ABI v2 稳定布局和有界递归值桥接。
- 增加协议与能力查询、应用/窗口资源、事件批次冲刷、整帧提交和窗口状态命令。
- 增加统一 CPU 二维渲染器，覆盖裁剪、变换、图形、路径、阴影、复杂文字和图片。
- 增加字体枚举、匹配、加载、文字整形、测量、字形位置、原文索引和命中测试。
- 增加有界计时器、图片解码、剪贴板、文件对话框、IME、光标、显示器与主题原语。
- 接通 `winit` 原生多窗口事件循环、`softbuffer` 表面、真实输入/IME/拖放事件与跨线程唤醒。
- 增加一次性结构化绘制树编译和正式二进制帧提交路径。
- 增加言序 1.1.7 公共包装 API、原生集成测试和自动退出真实窗口示例。
- 绘制协议提升至 v1.1，增加字族、字重、斜体文字和可重放字形序列。
- 让整形结果返回实际回退字体属性，并把加载字体同步到窗口渲染器。
- 最后一个窗口收到系统关闭时同时发送应用级退出请求。
- 升级复杂文字栈至 `cosmic-text 0.19`，移除停止维护的 RustyBuzz 链。
- 修补 Wayland 构建链的高危 `quick-xml` 公告，并增加六目标许可与安全审计策略。
- 增加平台、事件、绘制、文字、资源、线程、兼容、后端和架构决策文档。
